# Инструкция по работе с Git и GitHub

## Быстрый старт

### Основные команды для ежедневной работы

#### 1. Проверка статуса изменений
```bash
git status
```

#### 2. Просмотр изменений в файлах
```bash
git diff
```

#### 3. Добавление изменений в staging area
```bash
# Добавить все измененные файлы
git add .

# Или добавить конкретный файл
git add путь/к/файлу
```

#### 4. Создание коммита
```bash
git commit -m "Описание изменений"
```

Примеры хороших сообщений коммита:
- `git commit -m "Добавлена новая функция фильтрации"`
- `git commit -m "Исправлена ошибка в модуле авторизации"`
- `git commit -m "Обновлена документация"`

#### 5. Отправка изменений на GitHub
```bash
git push origin main
```

Если это первый push для новой ветки:
```bash
git push -u origin main
```

#### 6. Получение изменений с GitHub
```bash
git pull origin main
```

## Полный рабочий процесс

### Обновление и синхронизация проекта

#### Шаг 1: Получите последние изменения с GitHub


#### Шаг 3: Добавьте ваши изменения
```bash

```

#### Шаг 4: Создайте коммит
```bash
git commit -m "Ваше описание изменений"
```

#### Шаг 5: Отправьте на GitHub
```bash

```

## Полезные команды

### Просмотр истории коммитов
```bash
git log
```

### Просмотр истории в компактном виде
```bash
git log --oneline
```

### Отмена изменений в файле (до добавления в staging)
```bash
git checkout -- путь/к/файлу
```

### Отмена добавления файла из staging (но сохранение изменений)
```bash
git reset HEAD путь/к/файлу
```

### Просмотр информации о remote репозитории
```bash
git remote -v
```

### Проверка текущей ветки
```bash
git branch
```

## Решение проблем

### Если возникли конфликты при pull
```bash
# Git покажет конфликтующие файлы
# Отредактируйте их вручную, затем:
git add .
git commit -m "Разрешение конфликтов"
git push origin main
```

### Если нужно отменить последний коммит (но сохранить изменения)
```bash
git reset --soft HEAD~1
```

### Если нужно полностью отменить последний коммит
```bash
git reset --hard HEAD~1
```

⚠️ **Внимание:** `git reset --hard` удаляет все изменения безвозвратно!

## Типичный сценарий работы

1. **Начало работы:**
   ```bash
   git pull origin main  # Получить последние изменения
   ```

2. **Внесение изменений:**
   - Редактируйте файлы в проекте
   - Сохраняйте изменения

3. **Перед завершением работы:**
   ```bash
   git status           # Проверить что изменилось
   git add .            # Добавить изменения
   git commit -m "..."   # Создать коммит
   git push origin main  # Отправить на GitHub
   ```

## Примечания

- Всегда делайте `git pull` перед началом работы, чтобы получить последние изменения
- Делайте коммиты часто с понятными сообщениями
- Отправляйте изменения на GitHub регулярно, чтобы не потерять работу
- При работе в команде всегда синхронизируйтесь перед началом работы
