# Инструкция по настройке Git и синхронизации с GitHub

## Проблема
Из-за проблем с кодировкой пути (кириллица в имени пользователя) автоматическая настройка не работает. Выполните следующие шаги вручную.

## ⚠️ ВАЖНО: Сначала удалите неправильный .git

В домашней директории (`C:\Users\Ксения`) был создан неправильный Git репозиторий. Его нужно удалить:

1. Откройте Проводник Windows
2. Перейдите в `C:\Users\Ксения`
3. Включите отображение скрытых файлов (Вид → Показать → Скрытые элементы)
4. Удалите папку `.git` (если она там есть)

## Шаги для настройки:

### 1. Откройте терминал в директории проекта
Откройте PowerShell или командную строку и перейдите в директорию проекта:
```
cd "C:\Users\Ксения\OneDrive\Desktop\РМК\РТП-версии\2 версия\РТП-2.3.1\РТП-2.3"
```

### 2. Инициализируйте Git репозиторий
```bash
git init
```

### 3. Настройте Git с вашими данными
```bash
git config user.email "kkrivosheeva4@gmail.com"
git config user.name "kkrivosheeva4-tech"
```

### 4. Добавьте remote репозиторий
```bash
git remote add origin https://github.com/kkrivosheeva4-tech/RTP-version.git
```

### 5. Добавьте файлы проекта
```bash
git add .
```

### 6. Создайте первый коммит
```bash
git commit -m "Initial commit"
```

### 7. Отправьте код в GitHub
```bash
git push -u origin main
```

Если ветка называется `master` вместо `main`, используйте:
```bash
git branch -M main
git push -u origin main
```

## Альтернативный способ (использование готового скрипта)

Вы также можете использовать файл `init-git.bat`:
1. Откройте командную строку в директории проекта
2. Запустите: `init-git.bat`

Скрипт выполнит все шаги автоматически, кроме отправки в GitHub (её нужно будет выполнить вручную).

## Примечания

- Убедитесь, что у вас установлен Git: `git --version`
- Если репозиторий на GitHub еще не создан, создайте его на сайте GitHub
- При первом push может потребоваться аутентификация (токен доступа или пароль)
