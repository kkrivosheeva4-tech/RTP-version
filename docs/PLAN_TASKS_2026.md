## Задача 1. Валидация повторов технологий при создании/редактировании

> **Требования к валидации (применяются ко всем задачам валидации):**
> Система должна учитывать не только точное совпадение названий, но и похожие варианты ввода. Пользователи могут вводить данные по‑разному: с маленькой или заглавной буквы, с заменой русских букв на английские (например, `а`↔`a`, `о`↔`o`, `е`↔`e`), с лишними пробелами и другими вариациями. Валидация должна обрабатывать и проверять все возможные варианты, чтобы корректно определять дубликаты и предотвращать появление фактически одинаковых записей под разным написанием.

### Шаг 1.1 — Создать функцию валидации

**Файл:** `src/js/modules/core/validators.js` (новый) или добавить в существующий модуль

**Действия:**

- Добавить вспомогательную функцию `normalizeForComparison(str)` для переиспользования во всех валидациях: trim, `.toLowerCase()`, приведение омоглифов (а/a, о/o, е/e, р/p, у/y, с/c, х/x и т.д.) к единому виду.
- Добавить функцию `validateDuplicateTechnology(name, excludeId)`:
  - получает из `StateManager.get('technologies')` список технологий;
  - **нормализует** `name` для сравнения: trim, приведение к нижнему регистру, замена русско-английских омоглифов (а↔a, о↔o, е↔e, р↔p, у↔y и т.д.) в единый вид;
  - проверяет совпадение с уже существующими технологиями по нормализованному значению;
  - при `excludeId` — исключает технологию с этим ID (для режима редактирования);
  - возвращает `{ valid: boolean, message?: string }`.

**Пример сигнатуры:**

```javascript
function validateDuplicateTechnology(name, excludeId) {
  const technologies = StateManager.get("technologies") || [];
  const normalized = String(name || "")
    .trim()
    .toLowerCase();
  const duplicate = technologies.find((t) => {
    if (excludeId != null && t.id === excludeId) return false;
    return (
      String(t.name || "")
        .trim()
        .toLowerCase() === normalized
    );
  });
  return duplicate
    ? { valid: false, message: "Технология с таким названием уже существует" }
    : { valid: true };
}
```
