## 1. Начало. Описание проблемы и краткое описание решения

### Проблема

В современной цифровой трансформации крупных холдингов, таких как РМК, существует критическая необходимость в систематизации и визуализации технологического ландшафта. Основные вызовы:

- **Отсутствие единой картины**: Технологии разбросаны по разным предприятиям холдинга, нет централизованного представления
- **Сложность оценки готовности**: Нет объективных критериев для оценки готовности технологий к внедрению
- **Недостаток аналитики**: Сложно принимать стратегические решения без понимания текущего состояния цифровой зрелости
- **Отсутствие прозрачности**: Нет единой системы для отслеживания технологий, их статусов и взаимосвязей

### Решение

**«Радар технологий»** — интерактивная веб-платформа для визуализации и управления технологическим ландшафтом холдинга РМК.

**Ключевые преимущества решения:**

1. **Визуализация на радаре**: Технологии отображаются в виде интерактивного радара с квадрантами и кольцами готовности
2. **Математическая модель**: Объективная оценка готовности технологий на основе четырех факторов (технологическая готовность, организационная готовность, покрытие функций, TRL стадия)
3. **Многопользовательская система**: Разграничение прав доступа по ролям (гость, архитектор, директор, администратор)
4. **Централизованное управление**: Единая система для всех предприятий холдинга
5. **Аналитика и отчетность**: Встроенные инструменты для анализа и экспорта данных

## 2. Приложение. Главная страница. Шапка приложения

### Главная страница (index.html)

Главная страница приложения представляет собой точку входа в систему с визуализацией радара технологий.

**Основные элементы:**

1. **Шапка приложения (Header)**
   - **Логотип**: Иконка и название "Радар технологий" (кликабельно, ведет на главную)
   - **Кнопка помощи**: Доступ к справочной системе
   - **Кнопка уведомлений**: Показывает количество непрочитанных уведомлений (бейдж с числом)
   - **Переключатель темы**: Переключение между светлой и темной темой
   - **Информация об авторизации**: Отображает имя пользователя и роль (если авторизован)
   - **Кнопка выхода**: Выход из системы (для авторизованных пользователей)

2. **Основной контент**
   - **Заголовок**: "ИНТЕРАКТИВНАЯ ВИЗУАЛИЗАЦИЯ ЦИФРОВОЙ ЗРЕЛОСТИ"

**Особенности главной страницы:**

- Радар отображает все технологии без детальной информации
- При наведении на технологию показывается подсказка с названием

## 5. Страница радара. Обзор радара

### Структура радара

Радар представляет собой круговую диаграмму, разделенную на **4 квадранта**, каждый из которых соответствует направлению цифрового развития:

1. **Квадрант 1** (правый верхний): "Единый центр данных (Data Hub)"
2. **Квадрант 2** (левый верхний): "Искусственный интеллект во всех процессах"
3. **Квадрант 3** (левый нижний): "Автономные комплексы на всех предприятиях"
4. **Квадрант 4** (правый нижний): "Надёжная и безопасная цифровая инфраструктура"

### Элементы радара

#### 1. Технологии
- Отображаются в виде **кругов** на радаре
- **Размер круга** зависит от количества вендоров (чем больше вендоров, тем больше круг)
- **Позиция технологии** определяется:
  - **Квадрант**: по направлениям цифрового развития
  - **Расстояние от центра**: по математической модели готовности (чем ближе к центру, тем выше готовность)

#### 2. Зум (масштабирование)
- **Клик по квадранту**: Увеличивает масштаб выбранного квадранта
- **Правая панель технологий**: При зуме открывается панель со списком технологий в квадранте
- **Фильтрация в панели**: Можно фильтровать по статусу внедрения (Внедренные/Невнедренные)
- **Поиск в панели**: Поиск технологий по названию внутри квадранта

#### 3. Панель с технологиями (при зуме)
- Отображается справа при клике на квадрант
- Список технологий в квадранте
- Фильтры по статусу внедрения
- Поиск по названию
- Клик по технологии в списке подсвечивает её на радаре

#### 4. Детальная панель
Открывается при клике на технологию на радаре. Содержит:

- **Название технологии**
- **Функциональный блок**: Группа связанных функций
- **Функции**: Конкретные задачи, которые покрывает технология
- **TRL-стадия**: Уровень технологической готовности (1-Исследовательская, 2-Прототип, 3-Готова к внедрению)
- **Покрытие функций**: Оценка 0-3, рассчитывается автоматически
- **Оценки по предприятиям**: Вкладки с оценками технологической и организационной готовности для каждого предприятия
- **Описание**: Подробное описание технологии
- **Вендоры и интеграторы**: Список поставщиков и интеграторов
- **Примеры внедрений**: Реальные случаи использования
- **Прикрепленные файлы**: Документы, связанные с технологией

**Кнопки действий:**
- **Редактировать**: Открывает форму редактирования (для архитекторов и выше)
- **Удалить**: Удаление технологии с подтверждением (для архитекторов и выше)

#### 5. Расчет "Покрытия функций"

**Автоматический расчет:**

Покрытие функций (funcCover) рассчитывается автоматически на основе:

1. **Выбранных функций** технологии
2. **Функциональных блоков**, к которым относится технология
3. **Маппинга функций на блоки** (`functionToBlock.json`)
4. **Весов важности функций** (опционально, из `functionWeights.json`)

**Алгоритм расчета:**

```
1. Определение валидных функций (относящихся к блокам технологии)
2. Расчет взвешенной суммы всех функций в блоках
3. Расчет взвешенной суммы покрытых функций
4. Расчет процента покрытия = покрытые / все функции
5. Преобразование в оценку 0-3:
   - 0% → 0
   - <33% → 1 (низкое)
   - 33-67% → 2 (среднее)
   - ≥67% → 3 (высокое)
```

**Отображение:**
- Поле "Покрытие функций" в формах добавления/редактирования показывает рассчитанное значение
- Значение обновляется автоматически при изменении функций или блоков
- В детальной панели отображается визуально (звездочки или индикатор)

## 6. Математическая модель и как она устроена

### Общая концепция

Математическая модель позиционирования технологий использует **полярную систему координат** (θ, r):

- **θ (угол)** — определяет квадрант и позицию внутри квадранта
- **r (радиус)** — определяет "дистанцию до внедрения" (готовность технологии)

**Принцип:** Чем выше готовность технологии, тем **ближе к центру** она располагается (меньший радиус).

### Факторы готовности

Радиус рассчитывается на основе **четырех факторов готовности**:

1. **techRead** (0-3) — Технологическая готовность
   - Оценка зрелости технологии и доступности решений
   - 0 = не готова, 3 = полностью готова

2. **organRead** (0-3) — Организационная готовность
   - Готовность организации к внедрению технологии
   - 0 = не готова, 3 = полностью готова

3. **funcCover** (0-3) — Покрытие функций
   - Насколько полно технология покрывает необходимые функции
   - Рассчитывается автоматически (см. раздел 5)

4. **trlStage** (1-3) — TRL стадия (Technology Readiness Level)
   - 1 = Исследовательская
   - 2 = Прототип
   - 3 = Технология готова к внедрению

### Нормализация факторов

Все факторы нормализуются в диапазон [0, 1]:

```
x_techRead = techRead / 3
x_organRead = organRead / 3
x_funcCover = funcCover / 3
x_trlStage = (trlStage - 1) / 2
```

### Логистическая функция

**Сводный показатель готовности (z_i):**

Вычисляется как взвешенная сумма нормализованных факторов:

```
z_i = w_techRead * x_techRead + w_organRead * x_organRead +
      w_funcCover * x_funcCover + w_trlStage * x_trlStage
```

**Веса факторов (по умолчанию):**
- techRead: 30%
- organRead: 30%
- funcCover: 20%
- trlStage: 20%

**Важно:** Сумма весов всегда равна 1.0. Если фактор отсутствует, его вес перераспределяется между остальными.

### Расчет радиуса

**Формула:**

```
r_i = r_min + (r_max - r_min) * (1 - z_i)
```

где:
- `r_i` — радиус технологии в процентах (0 < r < 100)
- `r_min` — минимальный радиус (по умолчанию 5%)
- `r_max` — максимальный радиус (по умолчанию 95%)
- `z_i` — сводный показатель готовности (0 ≤ z_i ≤ 1)

**Интерпретация:**
- **Малый радиус (близко к центру)** = высокая готовность к внедрению
- **Большой радиус (близко к краю)** = низкая готовность к внедрению

### Расчет угла

**Определение квадранта:**

Квадрант определяется **только** на основе направлений цифрового развития технологии:

1. Технология имеет массив направлений (`tech.directions`)
2. Каждое направление привязано к квадрантам через маппинг `directionToQuadrant.json`
3. Технология отображается во всех квадрантах, соответствующих её направлениям

**Важно:** Функциональные блоки **не влияют** на определение квадранта. Они используются только для расчета покрытия функций.

**Формула расчета угла:**

```javascript
quadrantHash = tech.id * 1000 + quadrantId * 37
angleOffset = (quadrantHash * GOLDEN_ANGLE) % ANGLE_SPAN
theta = startAngle + ANGLE_PAD + angleOffset
```

где:
- `GOLDEN_ANGLE = 137.50776405003785°` — золотой угол для равномерного распределения
- `ANGLE_SPAN = 90 - (ANGLE_PAD * 2)` — доступный угловой диапазон квадранта
- `ANGLE_PAD = 8°` — отступ от границ квадранта
- `startAngle` — начальный угол квадранта (0°, 90°, 180°, 270°)

**Особенности:**
- Каждый квадрант получает уникальный угол на основе комбинации `tech.id` и `quadrantId`
- Углы детерминированы (стабильны при перезагрузке)
- Технологии распределяются равномерно по "лучам" от центра

### Распределение в квадранте

**Алгоритм распределения:**

1. Технологии сортируются по радиусу (от центра к краю)
2. Определяется оптимальное количество "лучей" (от 3 до 8)
3. Технологии распределяются по "лучам" циклически
4. Каждая технология получает небольшое детерминированное смещение (±1.5°) для избежания наложений

**Обработка отсутствующих данных:**

- Если отсутствует **более одного фактора** → технология позиционируется в зоне "недостаточно данных" (радиус 85%)
- Если отсутствует **один фактор** → его вес перераспределяется между остальными, расчет продолжается

### Пример расчета

**Дано:**
- techRead = 2.5
- organRead = 2.0
- funcCover = 2.0
- trlStage = 2

**Шаг 1: Нормализация**
```
x_techRead = 2.5 / 3 = 0.833
x_organRead = 2.0 / 3 = 0.667
x_funcCover = 2.0 / 3 = 0.667
x_trlStage = (2 - 1) / 2 = 0.5
```

**Шаг 2: Сводный показатель**
```
z_i = 0.30 * 0.833 + 0.30 * 0.667 + 0.20 * 0.667 + 0.20 * 0.5
z_i = 0.250 + 0.200 + 0.133 + 0.100 = 0.683
```

**Шаг 3: Радиус**
```
r_i = 5 + (95 - 5) * (1 - 0.683)
r_i = 5 + 90 * 0.317 = 33.53%
```

**Результат:** Технология позиционируется на расстоянии 33.53% от центра радара.

## 7. Работа с левой боковой панелью. Фильтрация и поиск

### Структура боковой панели

Левая боковая панель содержит основные инструменты для работы с радаром:

#### 1. Кнопки управления панелью

- **Кнопка показать/скрыть**: Сворачивание/разворачивание панели
- **Кнопка поиска**: Активация поля поиска
- **Кнопка фильтра**: Открытие панели фильтров
- **Кнопка сброса**: Сброс всех примененных фильтров
- **Кнопка отчета**: Экспорт данных в PDF (для авторизованных)
- **Кнопка добавления**: Добавление новой технологии или функционального блока (для архитекторов и выше)

#### 2. Поле поиска

**Функционал:**
- Поиск технологий по названию в реальном времени
- Автоматическое выделение найденных технологий на радаре
- Скрытие технологий, не соответствующих запросу
- Очистка поиска кнопкой или клавишей Escape

**Использование:**
- Введите название технологии в поле поиска
- Результаты фильтруются автоматически
- Для сброса очистите поле или нажмите кнопку "Сбросить выбор"

#### 3. Панель фильтров

**Доступные фильтры:**

1. **Предприятия** (множественный выбор)
   - Фильтрация по предприятиям холдинга
   - Показывает только технологии, относящиеся к выбранным предприятиям
   - Опция "Все предприятия"

2. **Функциональные блоки** (множественный выбор)
   - Фильтрация по группам связанных функций
   - Примеры: "Основное производство", "Корпоративное управление"
   - Можно выбрать несколько блоков одновременно

3. **Функции** (множественный выбор)
   - Фильтрация по конкретным функциям
   - Примеры: "Управление запасами", "Мониторинг оборудования"
   - Можно выбрать несколько функций

4. **Статус** (множественный выбор)
   - Фильтрация по статусу внедрения
   - Варианты: "Внедренные", "Невнедренные"
   - Можно выбрать оба статуса

**Применение фильтров:**

- Фильтры применяются автоматически при выборе значений
- Можно комбинировать несколько фильтров одновременно
- Технологии, не соответствующие фильтрам, скрываются на радаре
- В заголовке каждого фильтра показывается количество выбранных значений

**Сброс фильтров:**

- Кнопка "Сбросить выбор" очищает все примененные фильтры
- Также доступна в виде иконки в панели фильтров
- После сброса все технологии снова отображаются на радаре

### Взаимодействие фильтров и поиска

- **Поиск и фильтры работают совместно**: Технология должна соответствовать и поисковому запросу, и всем примененным фильтрам
- **Логика "И"**: Все условия должны выполняться одновременно
- **Сброс**: Кнопка "Сбросить выбор" очищает и фильтры, и поиск

## 8. Добавление технологий

### Доступ к функции

**Доступно для ролей:**
- Архитектор
- Директор
- Руководитель проекта
- Администратор

**Способы открытия формы:**

1. Кнопка "Добавить" в левой боковой панели
2. Всплывающее меню с выбором: "Добавить технологию" или "Добавить функциональный блок"

### Форма добавления технологии

Форма состоит из **вкладок**:

#### Вкладка "Общая информация"

**Обязательные поля:**

1. **Название** *
   - Краткое и понятное наименование технологии
   - Пример: "Система управления запасами"
   - Имеет подсказку с объяснением

2. **Направления цифрового развития** *
   - Определяет квадрант(ы) радара, где будет отображаться технология
   - Можно выбрать несколько направлений
   - Примеры: "Единый центр данных (Data Hub)", "Искусственный интеллект во всех процессах"
   - **Важно:** Это поле определяет позицию технологии на радаре

3. **Функциональный блок** *
   - Группа связанных функций
   - Используется для фильтрации (не влияет на квадрант)
   - Пример: "Основное производство"

4. **Функция** *
   - Конкретная задача или роль
   - Можно выбрать несколько функций
   - Пример: "Управление запасами"

5. **TRL (стадия готовности технологии)** *
   - 1-Исследовательская
   - 2-Прототип
   - 3-Технология готова к внедрению

6. **Покрытие функций**
   - Рассчитывается автоматически на основе выбранных функций
   - Поле только для чтения
   - Обновляется при изменении функций или блоков

7. **Описание** *
   - Краткое изложение сути технологии
   - Должно быть информативным

8. **Выбор предприятий** *
   - Вариант 1: Галочка "Технология применима в холдинге"
   - Вариант 2: Выбор конкретных предприятий из списка
   - Можно выбрать несколько предприятий

**Необязательные поля:**

1. **Примеры внедрений на рынке или в контуре**
   - Описание реальных случаев использования
   - Помогает понять практическую применимость

2. **Вендоры и интеграторы**
   - Выбор вендоров (поставщиков продукта)
   - Для каждого вендора можно указать интеграторов
   - Можно выбрать несколько вендоров

3. **Прикрепленные файлы**
   - Загрузка файлов с компьютера
   - Или добавление ссылок на файлы
   - Поддерживаются различные форматы

#### Вкладки предприятий

Для каждого выбранного предприятия создается отдельная вкладка, где можно указать:

- **Технологическая готовность** (0-3): Оценка зрелости технологии
- **Организационная готовность** (0-3): Готовность организации к внедрению

**Особенности:**
- Если выбрана галочка "Технология применима в холдинге", создается одна вкладка "Холдинг"
- Оценки влияют на расчет радиуса технологии на радаре

### Процесс добавления

1. **Заполнение формы**
   - Заполните все обязательные поля (отмечены звездочкой *)
   - При необходимости заполните дополнительные поля
   - Укажите оценки готовности для предприятий

2. **Автоматический расчет**
   - Система автоматически рассчитывает "Покрытие функций"
   - Позиция на радаре определяется автоматически по математической модели

3. **Сохранение**
   - Нажмите кнопку "Добавить технологию"
   - Система валидирует данные
   - При успешном сохранении:
     - Технология появляется на радаре
     - Открывается детальная панель с информацией о технологии
     - Создается уведомление об успешном добавлении
     - Действие записывается в журнал аудита

### Отображение на радаре

После добавления технология:

- **Появляется на радаре** в соответствующих квадрантах (по направлениям)
- **Позиционируется** по математической модели на основе оценок готовности
- **Размер круга** зависит от количества вендоров (если указаны)
- **Доступна для просмотра** всем пользователям (с соответствующими правами)

## 9. Редактирование технологии

### Доступ к функции

**Доступно для ролей:**
- Архитектор
- Директор

- Администратор

**Способы открытия формы:**

2. Через детальную панель технологии

### Форма редактирования

Форма редактирования **идентична** форме добавления, но:

- **Предзаполнена** текущими данными технологии
- **Заголовок**: "Редактировать технологию" (вместо "Добавить технологию")
- **Кнопка сохранения**: "Сохранить" (вместо "Добавить технологию")

### Структура формы

Форма имеет ту же структуру, что и форма добавления:

- **Вкладка "Общая информация"**: Все основные поля
- **Вкладки предприятий**: Оценки готовности для каждого предприятия

### Процесс редактирования

1. **Открытие формы**
   - Клик на технологию на радаре
   - Нажатие кнопки "Редактировать" в детальной панели
   - Форма открывается с предзаполненными данными

2. **Изменение данных**
   - Измените нужные поля
   - Все изменения валидируются в реальном времени
   - "Покрытие функций" пересчитывается автоматически

3. **Примеры изменений:**

   **Пример 1: Изменение оценок готовности**
   - Открываем технологию "Система управления запасами"
   - Переходим на вкладку предприятия "РМК"
   - Изменяем "Технологическая готовность" с 2.0 на 2.5
   - Изменяем "Организационная готовность" с 1.5 на 2.0
   - Сохраняем изменения

   **Результат:**
   - Технология перемещается ближе к центру радара (увеличилась готовность)
   - Обновляется детальная панель
   - Создается уведомление об успешном изменении
   - Действие записывается в журнал аудита

   **Пример 2: Добавление вендоров**
   - Открываем форму редактирования
   - В разделе "Вендоры и интеграторы" выбираем вендора "Microsoft"
   - Для вендора "Microsoft" выбираем интегратора "РМК-Диджитал"
   - Сохраняем изменения

   **Результат:**
   - Размер круга технологии увеличивается (больше вендоров)
   - В детальной панели отображаются вендоры и интеграторы
   - Обновляется информация о технологии

   **Пример 3: Изменение направлений**
   - Открываем форму редактирования
   - В поле "Направления цифрового развития" добавляем новое направление
   - Сохраняем изменения

   **Результат:**
   - Технология появляется в дополнительном квадранте
   - Позиция в новом квадранте рассчитывается автоматически
   - Технология может отображаться в нескольких квадрантах одновременно

4. **Сохранение**
   - Нажмите кнопку "Сохранить"
   - Система валидирует данные
   - При успешном сохранении:
     - Радар обновляется с новыми данными
     - Позиция технологии пересчитывается (если изменились оценки)
     - Детальная панель обновляется
     - Создается уведомление об успешном изменении
     - Действие записывается в журнал аудита

### Валидация

- Все обязательные поля должны быть заполнены
- Оценки готовности должны быть в диапазоне 0-3
- TRL стадия должна быть выбрана
- Направления цифрового развития должны быть указаны

### Отмена изменений

- Кнопка "Отмена" закрывает форму без сохранения
- При закрытии формы через крестик изменения не сохраняются
- Предупреждение не показывается (изменения не критичны)

## 11. Система уведомлений

### Обзор

Система уведомлений информирует пользователей о важных событиях и действиях в приложении.

### Элементы системы

#### 1. Кнопка уведомлений в шапке

- **Расположение**: В правом верхнем углу шапки приложения
- **Иконка**: Колокольчик с бейджем
- **Бейдж**: Показывает количество непрочитанных уведомлений (число)
- **Клик**: Открывает панель уведомлений

#### 2. Панель уведомлений

**Расположение**: Справа от экрана (выдвижная панель)

**Содержимое:**
- Список всех уведомлений
- Уведомления отсортированы по времени (новые сверху)
- Каждое уведомление содержит:
  - **Тип**: Иконка и цвет (успех, информация, предупреждение, ошибка)
  - **Заголовок**: Краткое описание события
  - **Сообщение**: Детальная информация
  - **Время**: Когда произошло событие
  - **Кнопка закрытия**: Удаление уведомления

### Типы уведомлений

1. **Успех (success)** — зеленый цвет
   - Успешное выполнение действия
   - Примеры: "Технология успешно добавлена", "Изменения сохранены"

2. **Информация (info)** — синий цвет
   - Информационные сообщения
   - Примеры: "Данные загружены", "Фильтры применены"

3. **Предупреждение (warning)** — желтый цвет
   - Предупреждения о потенциальных проблемах
   - Примеры: "Некоторые данные не загружены", "Недостаточно данных для расчета"

4. **Ошибка (error)** — красный цвет
   - Ошибки выполнения операций
   - Примеры: "Ошибка при сохранении", "Не удалось загрузить данные"

### События, фиксируемые в уведомлениях

#### Действия с технологиями

- ✅ **Добавление технологии**: "Технология '[Название]' успешно добавлена"
- ✅ **Редактирование технологии**: "Технология '[Название]' успешно обновлена"
- ✅ **Удаление технологии**: "Технология '[Название]' успешно удалена"
- ⚠️ **Ошибка сохранения**: "Ошибка при сохранении технологии '[Название]'"

#### Действия с функциональными блоками

- ✅ **Добавление блока**: "Функциональный блок '[Название]' успешно добавлен"
- ✅ **Редактирование блока**: "Функциональный блок '[Название]' успешно обновлен"
- ✅ **Удаление блока**: "Функциональный блок '[Название]' успешно удален"

#### Системные события

- ✅ **Авторизация**: "Успешный вход в систему"
- ✅ **Выход**: "Выход из системы"
- ℹ️ **Загрузка данных**: "Данные успешно загружены"
- ⚠️ **Ошибка загрузки**: "Ошибка при загрузке данных"

#### Экспорт и отчеты

- ✅ **Экспорт PDF**: "Отчет успешно экспортирован в PDF"
- ✅ **Экспорт данных**: "Данные успешно экспортированы"
- ⚠️ **Ошибка экспорта**: "Ошибка при экспорте данных"

### Интеграция с журналом аудита

**Все важные действия дублируются в журнале аудита:**

- Уведомление показывается пользователю
- Одновременно создается запись в журнале аудита
- Запись содержит:
  - Дата и время
  - Пользователь
  - Тип действия (create/update/delete)
  - Детали действия

**Пример:**

При добавлении технологии:
1. Показывается уведомление: "Технология 'Система управления запасами' успешно добавлена"
2. В журнале аудита создается запись:
   ```
   Дата: 2026-01-29 14:30:15
   Пользователь: architect
   Действие: create
   Детали: Технология 'Система управления запасами' добавлена
   ```

### Управление уведомлениями

- **Автоматическое скрытие**: Уведомления автоматически скрываются через 5 секунд (кроме постоянных)
- **Ручное закрытие**: Кнопка "×" в правом верхнем углу уведомления

### Форма добавления функционального блока

1. **Название блока** *
   - Краткое и понятное наименование функционального блока
   - Пример: "Основное производство"
   - Обязательное поле

2. **Направления** *

     - "Надёжная и безопасная цифровая инфраструктура"
   - Обязательное поле

3. **Добавление функций** *
   - Можно добавить до 20 функций

   - Кнопка "Добавить новую функцию" для добавления дополнительных полей

   - Фильтр: Выбор конкретных вендоров

    - Фильтр: Выбор конкретных интеграторов

    - Текстовое описание технологии

12. **Примеры внедрения**
    - Примеры использования технологии

1. **Выбор полей**
   - Отметьте нужные поля чекбоксами
   - Для полей с фильтрами настройте дополнительные параметры
   - Используйте "Выбрать все" для быстрого выбора всех полей

2. **Настройка фильтров** (опционально)
   - Для каждого поля с фильтром выберите нужные значения
   - Можно выбрать несколько значений (множественный выбор)
   - Фильтры применяются к экспортируемым данным

3. **Экспорт**
   - Нажмите кнопку "Экспортировать"
   - Система валидирует выбор (должно быть выбрано хотя бы одно поле)
   - Начинается процесс генерации PDF отчета

4. **Индикатор загрузки**
   - Показывается модальное окно с индикатором загрузки
   - Отображается прогресс генерации отчета
   - После завершения показывается сообщение об успехе

   - PDF файл автоматически скачивается
   - Имя файла: `radar_report_YYYY-MM-DD_HH-mm-ss.pdf`

   - Действие записывается в журнал аудита

- **Таблица данных**: Все выбранные поля в виде таблицы
- **Фильтры**: Информация о примененных фильтрах
- **Статистика**: Количество технологий в отчете

### Пример создания отчета

1. Нажимаем кнопку "Отчет" в боковой панели
2. В модальном окне выбираем поля:
   - ✓ Название технологии
   - ✓ Предприятия
   - ✓ Функциональный блок
   - ✓ Статус
   - ✓ Технологическая готовность
   - ✓ Организационная готовность
3. Настраиваем фильтры:
   - Предприятия: "РМК"
   - Статус: "Внедренные"
4. Нажимаем "Экспортировать"
5. Ждем завершения генерации (индикатор загрузки)
6. PDF файл скачивается автоматически
7. Появляется уведомление: "Отчет успешно экспортирован в PDF"
