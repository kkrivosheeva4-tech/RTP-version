## Общее описание приложения

«Радар технологий РМК» — это интерактивное веб-приложение для визуализации и анализа технологических решений и цифровой зрелости предприятий холдинга РМК. Приложение позволяет наглядно отслеживать технологический ландшафт организации, анализировать распределение и зрелость технологий, управлять данными и принимать обоснованные решения о внедрении новых технологических решений.

### Ключевые цели приложения

- **Визуализация технологического ландшафта** — представление всех технологий холдинга в едином интерактивном радаре
- **Анализ готовности к внедрению** — автоматический расчет готовности технологий на основе множественных факторов
- **Управление данными** — централизованное хранение и управление информацией о технологиях, предприятиях, вендорах
- **Поддержка принятия решений** — предоставление инструментов для анализа приоритетов и планирования внедрения

## Архитектура и структура

### Общая архитектура

Приложение построено на модульной архитектуре без использования серверной части (статическое веб-приложение). Все данные хранятся в JSON-файлах и localStorage браузера. Приложение состоит из шести основных страниц, каждая из которых решает определенные задачи.

### Структура данных

Данные организованы в следующих JSON-файлах:

- **enterpriseData.json** — основные данные о технологиях для стандартного просмотра
- **enterpriseData-director.json** — расширенные данные для директорской страницы (включая вендоров и файлы)
- **bloks.json** — справочник функциональных блоков
- **functions.json** — справочник функций
- **blockToQuadrant.json** — маппинг блоков к квадрантам радара
- **directionToQuadrant.json** — маппинг направлений цифрового развития к квадрантам
- **vendors.json** — справочник вендоров
- **integrators.json** — справочник интеграторов
- **technologies.json** — справочник типов технологий

### Модульная архитектура JavaScript

Приложение использует модульную систему, организованную по функциональным категориям:

- **Core модули** — управление состоянием, загрузка данных, обработка ошибок
- **UI модули** — компоненты интерфейса (фильтры, формы, модальные окна)
- **Radar модули** — визуализация и позиционирование технологий на радаре
- **Business модули** — бизнес-логика (авторизация, экспорт, расчет приоритетов)
- **Integration модули** — интеграция компонентов и обработка событий

## Описание страниц приложения

### 1. Главная страница (index.html → src/pages/index.html)

**Назначение:** Точка входа в приложение, лендинг с визуализацией радара и навигацией.

**Функциональность:**

- **Визуализация фонового радара** — отображение структуры радара с квадрантами и кольцами для демонстрации концепции
- **Навигация** — переход к детальному просмотру радара через кнопку "Перейти к детальному просмотру"
- **Общая навигация** — доступ к справке, переключение темы, авторизация

**Элементы интерфейса:**

- Заголовок с описанием назначения приложения
- Интерактивный SVG-радар в качестве фонового элемента
- Кнопка перехода к детальному просмотру
- Стандартная шапка с логотипом и элементами управления

**Особенности:**

- Страница служит визуальным введением в систему
- Не требует авторизации для просмотра
- Адаптирована для мобильных устройств

### 2. Страница авторизации (src/pages/auth.html)

**Назначение:** Безопасный вход в систему с разграничением прав доступа.

**Функциональность:**

- **Форма входа** — поля для имени пользователя и пароля
- **Запомнить меня** — сохранение сессии пользователя в localStorage
- **Вход как гость** — возможность просмотра без авторизации
- **Переключение темы** — поддержка светлой и темной темы
- **Валидация** — проверка обязательных полей и минимальной длины пароля
- **Показ/скрытие пароля** — кнопка для удобства ввода

**Элементы интерфейса:**

- Форма авторизации с полями:
  - Имя пользователя (обязательное)
  - Пароль (обязательное, минимум 4 символа)
  - Чекбокс "Запомнить меня"
  - Ссылка "Забыли пароль?"
- Кнопка "Войти"
- Кнопка "Войти как гость"
- Переключатель темы

**Безопасность:**

- Данные авторизации хранятся в localStorage (для демо-версии)
- Проверка прав доступа на каждой странице

**Роли пользователей:**

- **Гость** — просмотр без авторизации
- **Аналитик** — базовый доступ
- **Архитектор** — доступ к админ-панели
- **Директор/РП** — расширенный доступ к радару
- **Администратор** — полный доступ

### 3. Основная страница радара (src/pages/RMK-director.html)

**Назначение:** Главная рабочая страница приложения с интерактивным радаром технологий и полным функционалом управления.

**Функциональность:**

#### 3.1. Интерактивный радар

- **Визуализация технологий** — все технологии отображаются в виде кругов на радаре для единообразия визуализации
- **Позиционирование** — автоматическое размещение технологий на основе математической модели готовности
- **Квадранты** — четыре сектора, соответствующие направлениям цифрового развития:
  - Квадрант 1: Единый центр данных (Data Hub)
  - Квадрант 2: Искусственный интеллект во всех процессах
  - Квадрант 3: Автономные комплексы на всех предприятиях
  - Квадрант 4: Надёжная и безопасная цифровая инфраструктура
- **Кольца готовности** — расстояние от центра радара отражает готовность технологии к внедрению
- **Интерактивность**:
  - Клик по технологии открывает детальную панель
  - Наведение мыши показывает название технологии
  - Зум квадранта для детального просмотра
  - Фильтрация в реальном времени

#### 3.2. Боковая панель управления

**Поиск:**
- Текстовый поиск по названиям технологий
- Мгновенная фильтрация при вводе

**Фильтры:**
- **Предприятия** — мультиселект для выбора одного или нескольких предприятий
- **Функциональные блоки** — фильтрация по группам функций
- **Функции** — фильтрация по конкретным функциям (зависит от выбранных блоков)
- **Статус** — фильтрация по статусу внедрения (Внедренные, Невнедренные, Перспективные)

**Управление:**
- Кнопка сброса всех фильтров
- Кнопка экспорта в PDF (для авторизованных пользователей)
- Кнопка добавления новой технологии (для пользователей с правами)

#### 3.3. Детальная панель технологии

При клике на технологию открывается панель с подробной информацией:

- **Общая информация:**
  - Название технологии
  - Описание
  - Функциональный блок
  - Функции
  - TRL-стадия (Technology Readiness Level)
  - Покрытие функций (автоматический расчет)

- **Оценки по предприятиям:**
  - Вкладки для каждого предприятия, использующего технологию
  - Технологическая готовность (0-3)
  - Организационная готовность (0-3)
  - Приоритет внедрения (автоматический расчет)

- **Дополнительная информация:**
  - Вендоры и интеграторы
  - Примеры внедрений
  - Прикрепленные файлы

- **Действия:**
  - Редактирование (для авторизованных пользователей)
  - Удаление (для авторизованных пользователей)

#### 3.4. Панель приоритетов квадранта

При зуме квадранта открывается панель со списком технологий сектора:

- **Список технологий** — отсортированный по приоритету внедрения
- **Фильтры** — по статусу внедрения (Внедренные/Невнедренные)
- **Поиск** — по названиям технологий в списке
- **Синхронизация** — выделение технологии на радаре при выборе из списка

#### 3.5. Формы управления технологиями

**Добавление технологии:**

Форма с вкладками:
- **Общая информация:**
  - Название (обязательное)
  - Направления цифрового развития (обязательное, мультиселект)
  - Функциональный блок (обязательное)
  - Функция (обязательное)
  - TRL-стадия (обязательное)
  - Покрытие функций (автоматический расчет, только для чтения)
  - Описание (обязательное)
  - Выбор предприятий (обязательное)
  - Примеры внедрений (необязательное)
  - Вендоры и интеграторы (необязательное)
  - Прикрепленные файлы (необязательное)

- **Вкладки предприятий:**
  - Для каждого выбранного предприятия отдельная вкладка
  - Технологическая готовность (0-3)
  - Организационная готовность (0-3)

**Редактирование технологии:**

Аналогичная форма с предзаполненными данными.

**Особенности директорской страницы:**

- Размер фигур технологий зависит от количества вендоров:
  - 0-1 вендор: маленький размер (8px)
  - 2-3 вендора: средний размер (14px)
  - 4+ вендоров: большой размер (20px)
- Управление вендорами и интеграторами

## Математическая модель позиционирования

### Общая концепция

Позиционирование технологий на радаре основано на **полярной системе координат** с центром в точке (CENTER_X, CENTER_Y). Каждая технология позиционируется на основе двух параметров:

1. **Угол (θ)** — определяется принадлежностью технологии к направлению цифрового развития (квадранту)
2. **Радиус (r)** — вычисляется через математическую модель на основе факторов готовности

### Конвейер вычисления радиуса

Процесс вычисления радиуса проходит через следующие этапы:

```
Факторы готовности (s_ik)
  → Нормализация (x_ik)
  → Сводный показатель (z_i)
  → Логистическая функция (p_i)
  → Радиус (r_i)
```

### Факторы готовности

Модель использует четыре фактора готовности:

| Фактор | Диапазон | Описание |
|--------|----------|----------|
| `techRead` | 0-3 | Технологическая готовность (среднее по выбранным предприятиям) |
| `organRead` | 0-3 | Организационная готовность (среднее по выбранным предприятиям) |
| `funcCover` | 0-3 | Покрытие функций (общее значение для технологии) |
| `trlStage` | 1-3 | TRL стадия (Technology Readiness Level) |

**Особенности расчета:**
- Если выбраны предприятия в фильтре, вычисляется среднее значение только по выбранным предприятиям
- Если фильтр не применен, используются все предприятия технологии
- Если у технологии нет данных по предприятиям, используются общие значения (fallback)

### Нормализация факторов

Все факторы нормализуются в диапазон [0, 1]:

```
x_techRead = techRead / 3        // 0-3 → 0-1
x_organRead = organRead / 3      // 0-3 → 0-1
x_funcCover = funcCover / 3      // 0-3 → 0-1
x_trlStage = (trlStage - 1) / 2  // 1-3 → 0-1
```

**Обработка отсутствующих данных:**
- Если `techRead` или `organRead` отсутствуют (null/undefined), используется среднее значение 0.5 вместо 0
- Это позволяет различать "отсутствие данных" и "минимальная готовность"

### Сводный показатель

Вычисляется как взвешенная сумма нормализованных факторов плюс смещение (bias):

```
z_i = w_techRead * x_techRead +
      w_organRead * x_organRead +
      w_funcCover * x_funcCover +
      w_trlStage * x_trlStage +
      bias
```

**Веса факторов (по умолчанию):**
- `w_techRead = 0.30` (30%)
- `w_organRead = 0.30` (30%)
- `w_funcCover = 0.20` (20%)
- `w_trlStage = 0.20` (20%)
- **Сумма весов = 1.0** (автоматическая нормализация при переопределении)

**Смещение (bias):**
- По умолчанию: `bias = -0.6`
- Калибровка модели:
  - Максимальные параметры (все = 3) → r ≈ 17% (близко к центру)
  - Средние параметры (все = 1.5) → r ≈ 60% (среднее положение)
  - Низкие параметры (все = 0) → r ≈ 92% (у края)

**Конфигурируемость:**
Параметры модели можно переопределить через `window.RadarModelConfig`:
- `alpha` — параметр чувствительности логистической функции (по умолчанию: 4)
- `bias` — смещение для калибровки (по умолчанию: -0.6)
- `weights` — веса факторов (по умолчанию: techRead=0.30, organRead=0.30, funcCover=0.20, trlStage=0.20)

### Логистическая функция

Преобразует сводный показатель в вероятность готовности:

```
p_i = 1 / (1 + exp(-α * z_i))
```

где:
- `α` (alpha) = 4 (по умолчанию) — параметр чувствительности
- `p_i ∈ (0, 1)` — степень близости к внедрению (вероятность готовности)

**Свойства логистической функции:**
- При `z_i → +∞`: `p_i → 1` (высокая готовность)
- При `z_i → -∞`: `p_i → 0` (низкая готовность)
- При `z_i = 0`: `p_i = 0.5` (средняя готовность)

### Вычисление радиуса

Радиус вычисляется как инверсия вероятности готовности:

```
r_i = 100 * (1 - p_i)
```

**Интерпретация:**
- `p_i = 1` (максимальная готовность) → `r_i = 0` (центр радара)
- `p_i = 0` (минимальная готовность) → `r_i = 100` (край радара)
- `p_i = 0.5` (средняя готовность) → `r_i = 50` (середина)

**Гарантия диапазона:**
Радиус гарантированно находится в диапазоне `(0, 100)`:
- Если `r_i ≤ 0`: устанавливается `r_i = 0.01` (EPSILON)
- Если `r_i ≥ 100`: устанавливается `r_i = 99.99`

### Вычисление угла

Угол определяется принадлежностью технологии к квадранту:

1. **Определение квадранта**: По первому направлению технологии через маппинг `directionToQuadrant`

2. **Базовый угол квадранта**: Каждый квадрант имеет `startAngle`:
   - Квадрант 1: 0°
   - Квадрант 2: 90°
   - Квадрант 3: 180°
   - Квадрант 4: 270°

3. **Распределение внутри квадранта**: Используется золотой угол (Golden Angle = 137.50776405003785°) для равномерного распределения:

```
ANGLE_PAD = 8°  // Отступ от границ квадранта
ANGLE_SPAN = 90 - (ANGLE_PAD * 2) = 74°  // Доступный диапазон
angleOffset = (tech.id * GOLDEN_ANGLE) % ANGLE_SPAN
theta = startAngle + ANGLE_PAD + angleOffset
```

### Преобразование в декартовы координаты

После вычисления полярных координат (θ, r), они преобразуются в декартовы:

```
// Масштабирование радиуса из процентов (0-100) в пиксели
maxR = количество_колец * RADIUS_STEP
availableRadius = maxR - POSITION_PAD
radius_pixels = POSITION_PAD + (r_i / 100) * availableRadius

// Преобразование полярных координат в декартовы
x = CENTER_X + radius_pixels * cos(θ)
y = CENTER_Y + radius_pixels * sin(θ)
```

**Константы:**
- `CENTER_X = 500` (пикселей)
- `CENTER_Y = 500` (пикселей)
- `RADIUS_STEP = 140` (пикселей между кольцами)
- `POSITION_PAD = 30` (пикселей отступ от границ)

### Расчет оценки "Покрытие функций"

Оценка "Покрытие функций" (`funcCover`) вычисляется на основе процентного покрытия функциональных блоков технологии.

**Алгоритм расчета:**

1. **Определение покрытых функций**: Технология имеет массив `functionCoverage` — список функций, которые она покрывает

2. **Определение блоков технологии**: Технология имеет массив `blocks` — список функциональных блоков, к которым она относится

3. **Подсчет функций в блоках**: Для каждого блока технологии определяется общее количество функций через маппинг `functionToBlock.json`

4. **Учет важности функций (опционально)**: Если включен учет важности (`useWeights: true`), используется взвешенное покрытие:
   - Загрузка весов из `functionWeights.json`
   - Взвешенная сумма всех функций в блоках
   - Взвешенная сумма покрытых функций

5. **Вычисление процента покрытия**:
   ```
   coveragePercent = coveredWeightedSum / totalWeightedFunctions
   ```

6. **Преобразование процента в оценку funcCover (0-3)**:
   - 0% → 0 (не покрывает необходимые функции)
   - 1-33% → 1 (низкое покрытие функций)
   - 34-67% → 2 (среднее покрытие функций)
   - 68-100% → 3 (высокое покрытие функций)

**Особенности:**
- Если есть хотя бы одна функция, `funcCover` минимум = 1
- Используется плавное округление с порогом 5% для избежания резких переходов

### Функция контроля наложений технологий

Для предотвращения наложения технологий друг на друга используется итеративный алгоритм разведения.

**Общая структура алгоритма:**

1. Группировка по квадрантам
2. Равномерное распределение по углу
3. Ограничение в пределах квадранта
4. Предварительное разведение идентичных координат
5. Итеративное разведение близких точек
6. Финальное разведение по радиусу (если необходимо)

**Метод отталкивания:**

Используется итеративный алгоритм с параметрами:
- `MAX_ITER = 80-150` (адаптивно в зависимости от количества технологий)
- `CONVERGENCE_THRESHOLD = 0.1px`
- `DAMPING_DECAY = 0.98` (коэффициент затухания)

**Особенности алгоритма:**
- Адаптивное минимальное расстояние в зависимости от количества технологий в квадранте
- Учет размеров элементов: минимальное расстояние = сумма радиусов + зазор 4px
- Коэффициент затухания уменьшается с каждой итерацией для стабильности
- Усиленное отталкивание для очень близких точек (множитель 1.5)

**Оптимизация для больших наборов данных:**

Если в квадранте более 20 технологий и доступен модуль `SpatialIndex`, используется оптимизированный алгоритм с пространственными индексами.

## Система ролей и прав доступа

### Гость (guest)

**Права доступа:**
- Просмотр радара технологий
- Просмотр детальной информации о технологиях
- Использование фильтров и поиска
- Просмотр справки

**Ограничения:**
- Нет доступа к административной панели
- Нет возможности добавлять/редактировать технологии
- Нет доступа к экспорту данных

### Аналитик (analyst)

**Права доступа:**
- Все права гостя
- Расширенный просмотр данных
- Базовый анализ технологий

**Ограничения:**
- Нет доступа к административной панели
- Нет возможности модифицировать данные

### Архитектор (architect)

**Права доступа:**
- Все права аналитика
- Доступ к административной панели
- Управление пользователями
- Просмотр журнала аудита
- Управление предприятиями
- Добавление и редактирование технологий
- Экспорт данных

**Ограничения:**
- Нет доступа к резервным копиям (только для администраторов)

### Директор / Руководитель проекта (director / project_manager)

**Права доступа:**
- Все права архитектора
- Доступ к расширенной странице радара (RMK-director.html)
- Управление вендорами и интеграторами
- Прикрепление файлов к технологиям
- Управление приоритетами секторов
- Просмотр модифицированного радара с позиционированием по готовности

**Особенности:**
- Работа с отдельным файлом данных (enterpriseData-director.json)
- Расширенные формы с дополнительными полями

### Администратор (admin)

**Права доступа:**
- Полный доступ ко всем функциям системы
- Все права директора
- Управление резервными копиями
- Сброс локальных правок данных
- Полный доступ к экспорту всех данных
- Управление всеми пользователями
