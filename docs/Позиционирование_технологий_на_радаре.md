# Позиционирование технологий на радаре

## Обзор

Технологический радар представляет собой круговую диаграмму, на которой технологии позиционируются в полярных координатах (угол θ и радиус r). Позиция каждой технологии определяется математической моделью на основе её характеристик готовности и принадлежности к направлениям цифрового развития.

## Структура радара

### Квадранты

Радар разделен на 4 квадранта, соответствующих направлениям цифрового развития:

1. **Квадрант 1**: "Единый центр данных (Data Hub)"
2. **Квадрант 2**: "Искусственный интеллект во всех процессах"
3. **Квадрант 3**: "Автономные комплексы на всех предприятиях"
4. **Квадрант 4**: "Надёжная и безопасная цифровая инфраструктура"

### Кольца

Радар состоит из трёх концентрических колец:
- **Внешнее кольцо**: Технологии, наиболее удалённые от внедрения
- **Среднее кольцо**: Технологии со средней готовностью
- **Внутреннее кольцо (центр)**: Технологии, наиболее близкие к внедрению

## Математическая модель позиционирования

### Вычисление угла (θ)

Угол определяется принадлежностью технологии к направлению цифрового развития:

1. **Определение квадранта**: Технология относится к квадранту на основе поля `directions` (массив ID направлений)
2. **Базовый угол**: Каждый квадрант имеет начальный угол (`startAngle`) и занимает 90° сектор
3. **Распределение внутри квадранта**:
   - Используется золотой угол (137.50776405003785°) для равномерного распределения
   - Угол вычисляется как: `θ = startAngle + ANGLE_PAD + (id × GOLDEN_ANGLE) % ANGLE_SPAN`
   - `ANGLE_PAD` = 8° (отступ от границ квадранта)
   - `ANGLE_SPAN` = 90° - (2 × ANGLE_PAD) = 74° (доступный диапазон)

**Формула угла:**
```
θ = q.startAngle + POSITION_ANGLE_PAD + ((tech.id × GOLDEN_ANGLE) % ANGLE_SPAN)
```

### Вычисление радиуса (r)

Радиус вычисляется как "дистанция до внедрения" на основе математической модели с логистической функцией.

#### Шаг 1: Извлечение факторов готовности

Для каждой технологии учитываются следующие факторы:

- **techRead** (0-3): Технологическая готовность - **среднее значение** `technologicalReadiness` по всем предприятиям из массива `enterprises`
- **organRead** (0-3): Организационная готовность - **среднее значение** `organizationalReadiness` по всем предприятиям из массива `enterprises`
- **funcCover** (0-3): Покрытие функций - **общее значение** для технологии (не зависит от предприятия)
- **trlStage** (1-3): Технологический уровень готовности (TRL) - **общее значение** для технологии (не зависит от предприятия)

**Важно:**
- Оценки `techRead` и `organRead` **не хранятся** как общие значения технологии
- Эти оценки выставляются **только для предприятий** в массиве `enterprises`
- Если технология применима для нескольких предприятий с разными оценками, вычисляется **среднее арифметическое** значение
- `funcCover` и `trlStage` являются **общими** для всех предприятий

#### Шаг 2: Нормализация факторов

Все факторы нормализуются в диапазон [0, 1]:

```
x_techRead = techRead / 3
x_organRead = organRead / 3
x_funcCover = funcCover / 3
x_trlStage = (trlStage - 1) / 2
```

#### Шаг 3: Вычисление сводного показателя

Сводный показатель вычисляется как взвешенная сумма нормализованных факторов:

```
z_i = Σ(w_k × x_ik) + b
```

**Веса факторов:**
- `w_techRead = 0.25` (25%)
- `w_organRead = 0.25` (25%)
- `w_funcCover = 0.15` (15%)
- `w_trlStage = 0.20` (20%)

**Сдвиг (bias):**
- `b = -0.5` (отрицательный сдвиг делает модель более строгой)

**Формула:**
```
z_i = 0.25 × x_techRead + 0.25 × x_organRead + 0.15 × x_funcCover + 0.20 × x_trlStage - 0.5
```

#### Шаг 4: Логистическая функция

Сводный показатель преобразуется через логистическую функцию:

```
p_i = 1 / (1 + exp(-α × z_i))
```

где:
- `α = 4` (параметр чувствительности логистической функции)
- `p_i ∈ (0, 1)` - степень близости к внедрению (вероятность готовности)

#### Шаг 5: Вычисление радиуса

Радиус вычисляется как инверсия вероятности готовности:

```
r_i = 100 × (1 - p_i)
```

**Интерпретация:**
- Чем выше `p_i` (ближе к 1), тем меньше радиус (ближе к центру)
- Чем ниже `p_i` (ближе к 0), тем больше радиус (дальше от центра)

#### Шаг 6: Граничные условия

Радиус гарантированно находится в диапазоне `(0, 100)` с минимальным отступом `EPSILON = 0.01` от границ:

```
r_i = clamp(r_i, EPSILON, 100 - EPSILON)
```

**Примечание:** Правило о минимальном количестве вендоров для размещения в центральном круге **удалено**. Технологии позиционируются исключительно на основе математической модели готовности.

#### Шаг 7: Масштабирование к координатам SVG

Радиус в процентах (0-100) масштабируется к реальным координатам SVG:

```
maxR = RINGS.length × RADIUS_STEP
availableRadius = maxR - POSITION_PAD
radius_pixels = POSITION_PAD + (r_i / 100) × availableRadius
```

где:
- `RADIUS_STEP = 140` пикселей (расстояние между кольцами)
- `POSITION_PAD = 30` пикселей (отступ от центра)

## Преобразование в декартовы координаты

После вычисления полярных координат (θ, r) они преобразуются в декартовы координаты (x, y):

```
x = CENTER_X + r × cos(θ × π / 180)
y = CENTER_Y + r × sin(θ × π / 180)
```

где:
- `CENTER_X = 500` (центр радара по X)
- `CENTER_Y = 500` (центр радара по Y)

## Разведение технологий (Collision Avoidance)

Для предотвращения наложения технологий применяется алгоритм разведения:

### Группировка

Технологии группируются по квадрантам для независимой обработки каждого сектора.

### Алгоритм разведения

1. **Предварительное разведение**: Если технологии имеют идентичные координаты, они слегка разводятся по углу на основе их ID
2. **Итеративное разведение**:
   - Выполняется до 120 итераций
   - Для каждой пары технологий проверяется расстояние между ними
   - Минимальное расстояние рассчитывается как сумма радиусов элементов плюс зазор (4px)
   - Если технологии слишком близко, они отталкиваются друг от друга
3. **Адаптивное расстояние**:
   - Базовое минимальное расстояние: `MIN_BLIP_DISTANCE = 28px`
   - Для групп с более чем 10 технологиями: увеличивается на 30%
   - Для групп с более чем 5 технологиями: увеличивается на 15%
4. **Ограничение границами**: После каждого перемещения координаты ограничиваются границами квадранта

### Финальная проверка

Если после всех итераций остались наложения, применяется радикальное разведение по радиусу: одна технология перемещается ближе к центру, другая - дальше от центра.

## Оценки для предприятий

Технология может иметь поле `enterprises` с массивом объектов, содержащих оценки для каждого предприятия:

```json
{
  "enterprises": [
    {
      "enterpriseId": 1,
      "technologicalReadiness": 3,
      "organizationalReadiness": 2,
      "status": "Внедрена"
    },
    {
      "enterpriseId": 2,
      "technologicalReadiness": 2,
      "organizationalReadiness": 3,
      "status": "Невнедренна"
    }
  ]
}
```

**Логика вычисления:**
- Если технология применима для нескольких предприятий с разными оценками, вычисляется **среднее арифметическое** значение `technologicalReadiness` и `organizationalReadiness` по всем предприятиям
- Общие значения `techRead` и `organRead` **не используются** - эти оценки выставляются только для предприятий
- `funcCover` и `trlStage` остаются **общими** для всех предприятий

**Пример расчета:**
- Предприятие 1: `technologicalReadiness = 3`, `organizationalReadiness = 2`
- Предприятие 2: `technologicalReadiness = 2`, `organizationalReadiness = 3`
- **Среднее:** `techRead = (3 + 2) / 2 = 2.5`, `organRead = (2 + 3) / 2 = 2.5`

## Примеры

### Пример 1: Технология с высокой готовностью

**Данные:**
- `enterprises = [{technologicalReadiness: 3, organizationalReadiness: 3}, {technologicalReadiness: 3, organizationalReadiness: 3}]`
- Среднее: `techRead = 3`, `organRead = 3`
- `funcCover = 3` (общее значение)
- `trlStage = 3` (общее значение)
- `directions = [1]` (Квадрант 1)

**Расчет:**
1. Нормализация: `x_techRead = 1.0`, `x_organRead = 1.0`, `x_funcCover = 1.0`, `x_trlStage = 1.0`
2. Сводный показатель: `z_i = 0.25 + 0.25 + 0.15 + 0.20 - 0.5 = 0.35`
3. Логистическая функция: `p_i = 1 / (1 + exp(-4 × 0.35)) ≈ 0.80`
4. Радиус: `r_i = 100 × (1 - 0.80) = 20%`
5. Угол: определяется квадрантом 1 и ID технологии

**Результат**: Технология размещается близко к центру радара в квадранте 1 (независимо от количества вендоров).

### Пример 2: Технология с низкой готовностью

**Данные:**
- `enterprises = [{technologicalReadiness: 1, organizationalReadiness: 1}]`
- Среднее: `techRead = 1`, `organRead = 1`
- `funcCover = 1` (общее значение)
- `trlStage = 1` (общее значение)
- `directions = [3]` (Квадрант 3)

**Расчет:**
1. Нормализация: `x_techRead = 0.33`, `x_organRead = 0.33`, `x_funcCover = 0.33`, `x_trlStage = 0.0`
2. Сводный показатель: `z_i = 0.25×0.33 + 0.25×0.33 + 0.15×0.33 + 0.20×0.0 - 0.5 ≈ -0.28`
3. Логистическая функция: `p_i = 1 / (1 + exp(-4 × (-0.28))) ≈ 0.25`
4. Радиус: `r_i = 100 × (1 - 0.25) = 75%`
5. Угол: определяется квадрантом 3 и ID технологии

**Результат**: Технология размещается на внешнем краю радара в квадранте 3.

### Пример 3: Технология с несколькими предприятиями (разные оценки)

**Данные:**
- `enterprises = [
    {technologicalReadiness: 3, organizationalReadiness: 2},
    {technologicalReadiness: 2, organizationalReadiness: 3},
    {technologicalReadiness: 1, organizationalReadiness: 1}
  ]`
- Среднее: `techRead = (3 + 2 + 1) / 3 = 2.0`, `organRead = (2 + 3 + 1) / 3 = 2.0`
- `funcCover = 2` (общее значение)
- `trlStage = 2` (общее значение)
- `directions = [2]` (Квадрант 2)

**Расчет:**
1. Нормализация: `x_techRead = 2.0/3 ≈ 0.67`, `x_organRead = 2.0/3 ≈ 0.67`, `x_funcCover = 2/3 ≈ 0.67`, `x_trlStage = (2-1)/2 = 0.5`
2. Сводный показатель: `z_i = 0.25×0.67 + 0.25×0.67 + 0.15×0.67 + 0.20×0.5 - 0.5 ≈ -0.08`
3. Логистическая функция: `p_i = 1 / (1 + exp(-4 × (-0.08))) ≈ 0.42`
4. Радиус: `r_i = 100 × (1 - 0.42) = 58%`
5. Угол: определяется квадрантом 2 и ID технологии

**Результат**: Технология размещается в среднем кольце радара в квадранте 2.

## Зависимости и конфигурация

### Глобальные переменные

Позиционирование использует следующие глобальные переменные:

- `QUADRANTS`: Массив квадрантов с полями `id`, `name`, `startAngle`
- `RINGS`: Массив названий колец
- `CENTER_X`, `CENTER_Y`: Координаты центра радара
- `RADIUS_STEP`: Расстояние между кольцами (140px)
- `POSITION_PAD`: Отступ от центра (30px)
- `POSITION_ANGLE_PAD`: Отступ от границ квадранта (8°)
- `MIN_BLIP_DISTANCE`: Минимальное расстояние между технологиями (28px)
- `directionToQuadrant`: Маппинг направлений на квадранты

### Файлы данных

- `src/data/ru/technologies.json`: Данные о технологиях
- `src/data/ru/digitalDirections.json`: Направления цифрового развития
- `src/data/ru/directionToQuadrant.json`: Маппинг направлений на квадранты

### Модули

- `src/js/modules/radar/positioning.js`: Основной модуль позиционирования
- `src/js/modules/radar/radar-renderer.js`: Модуль отрисовки радара
- `src/js/modules/radar/quadrants.js`: Модуль работы с квадрантами

## Заключение

Позиционирование технологий на радаре основано на математической модели, которая учитывает:

1. **Принадлежность к направлению** (определяет квадрант и угол)
2. **Уровень готовности** (определяет радиус через логистическую функцию):
   - **Средние значения** технологической и организационной готовности по всем предприятиям
   - **Общие значения** покрытия функций и TRL стадии
3. **Равномерное распределение** (алгоритм разведения предотвращает наложения)

Модель обеспечивает объективное и воспроизводимое позиционирование технологий, отражающее их реальную готовность к внедрению.
