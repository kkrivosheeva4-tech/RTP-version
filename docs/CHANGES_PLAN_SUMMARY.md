# Краткое резюме плана изменений

## Основные изменения

### 0. Переход на директорскую страницу как основную
- Сделать `RMK-director.html` единственной основной страницей
- Удалить или оставить `RMK.html` только для редиректа
- Все роли (архитекторы, директоры, РП, администраторы) работают на одной странице
- Обновить все редиректы и ссылки

### 1. Деление радара по направлениям цифрового развития
- **Ключевое изменение:** Направления определяют квадранты (позицию на радаре)
- **Важно:** Функциональные блоки НЕ связаны с секторами/квадрантами
- Функциональные блоки используются только для фильтрации и классификации
- **4 новых квадранта (направления):**
  1. Единый центр данных (Data Hub) - Квадрант 1
  2. Искусственный интеллект во всех процессах - Квадрант 2
  3. Автономные комплексы на всех предприятиях - Квадрант 3
  4. Надёжная и безопасная цифровая инфраструктура - Квадрант 4
- Обновить логику позиционирования: угол θ определяется направлением, а не блоком

### 2. Предприятия в левой панели
- Убрать предприятия из шапки
- Добавить фильтр предприятий в левую панель
- Объединить данные всех предприятий в один радар
- Реализовать фильтрацию технологий по предприятиям

### 3. Каскадная фильтрация
- При выборе предприятия показывать только относящиеся к нему блоки и функции
- Реализовать зависимость: Предприятия → Блоки → Функции
- **Важно:** Для каждого предприятия могут быть свои функциональные блоки
- Некоторые функциональные блоки могут не быть ни в одном предприятии (общие блоки)

### 4. Карточка добавления технологии
**Обязательные поля:**
- Название технологии
- Направления цифрового развития (новое, определяет позицию на радаре)
- Функциональный блок (используется только для фильтрации)
- Функция
- Описание

**Необязательные поля:**
- Предприятия (необязательное, множественный выбор)
- Примеры внедрения
- Вендоры и интеграторы
- Прикрепленные файлы

**Важно:**
- Поля для каждого предприятия (технологическая готовность, организационная готовность, TRL стадия, внедрена/не внедрена) **НЕ отображаются** в общей карточке добавления
- Эти поля заполняются отдельно после добавления технологии (в карточке редактирования)

**Новые функции:**
- Автоматический расчет "Покрытие функций" на основе количества выбранных функций
- Ограничения по размерам файлов
- Заполнение полей для предприятий в карточке редактирования (после добавления)

### 5. Удаление полей из карточек
- Убрать "Тип технологий" и "Статус" из карточки редактирования
- Убрать из карточки просмотра (или оставить только для чтения)

### 6. Управление предприятиями (для администратора)
- Карточка для добавления/редактирования/удаления предприятий
- Доступна только для роли `admin`

## 4 направления цифрового развития (новые квадранты)

1. **Единый центр данных (Data Hub)** - Квадрант 1 (0°–90°)
   - Единая точка управления всей компанией в реальном времени и «одна версия правды» для всех процессов и подразделений

2. **Искусственный интеллект во всех процессах** - Квадрант 2 (90°–180°)
   - Широкое внедрение ИИ-решений на всех уровнях деятельности компании

3. **Автономные комплексы на всех предприятиях** - Квадрант 3 (180°–270°)
   - Создание «light out» (безлюдных или минимально обслуживаемых) участков и предприятий, функционирующих в автономном режиме

4. **Надёжная и безопасная цифровая инфраструктура** - Квадрант 4 (270°–360°)
   - Необходима для масштабирования решений на основе искусственного интеллекта и автономных систем, обеспечивая устойчивость, безопасность и производительность
