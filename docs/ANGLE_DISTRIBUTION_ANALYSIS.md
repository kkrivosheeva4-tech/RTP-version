## Описание проблемы

### Текущая ситуация

В текущей реализации технологии в квадранте распределяются **равномерно по углу** независимо от их радиуса (расстояния от центра). Это означает:

- Технология A с радиусом 50px имеет угол 45°
- Технология B с радиусом 150px имеет угол 50°
- Технология C с радиусом 250px имеет угол 55°

**Визуальный результат:** Технологии разбросаны по квадранту, образуя "облако" точек без явной структуры.

### Проблема

1. **Визуальная хаотичность:** Технологии с разными радиусами имеют разные углы, что создает впечатление случайного распределения
2. **Сложность восприятия:** Пользователю сложно визуально группировать технологии по готовности (радиусу)
3. **Потеря структуры:** Нет явной связи между технологиями на разных радиусах

### Вопрос

**Логично ли размещать технологии с разными радиусами под одним углом, создавая "лучи" от центра?**

## Текущий подход

### Принцип работы

**Равномерное распределение по углу (Uniform Angle Distribution)**

```javascript
// Текущая реализация
group.forEach((t, index) => {
  const ratio = index / (group.length - 1);
  targetAngle = angleMin + ratio * availableAngleSpan;
  // Радиус сохраняется из математической модели
  newPos = polarToCartesian(radius, targetAngle);
});
```

**Характеристики:**
- Каждая технология получает уникальный угол
- Углы равномерно распределены по доступному диапазону квадранта (74°)
- Радиус определяется математической моделью готовности
- Угол и радиус независимы друг от друга

**Визуальный результат:**
```
        ┌─────────────────┐
        │  ●              │  Технология 1 (r=50, θ=45°)
        │     ●           │  Технология 2 (r=150, θ=50°)
        │        ●        │  Технология 3 (r=250, θ=55°)
        │           ●     │  Технология 4 (r=100, θ=60°)
        └─────────────────┘
```

### Преимущества текущего подхода

1. **Максимальное использование пространства:** Квадрант заполняется равномерно
2. **Минимизация наложений:** При небольшом количестве технологий наложения минимальны
3. **Простота реализации:** Алгоритм прост и предсказуем
4. **Сохранение семантики радиуса:** Радиус (готовность) не искажается

### Недостатки текущего подхода

1. **Визуальная хаотичность:** Технологии выглядят разбросанными
2. **Сложность группировки:** Сложно визуально группировать технологии по готовности
3. **Отсутствие структуры:** Нет явной визуальной структуры "лучей"
4. **Сложность сравнения:** Сложнее сравнивать технологии на разных радиусах

## Альтернативные подходы

### Подход 1: Группировка по углам ("Лучи")

**Принцип:** Технологии группируются по фиксированным углам, образуя "лучи" от центра.

```javascript
// Псевдокод
const fixedAngles = calculateFixedAngles(group.length, angleMin, angleMax);
group.forEach((t, index) => {
  targetAngle = fixedAngles[index % fixedAngles.length];
  // Радиус сохраняется из математической модели
  newPos = polarToCartesian(radius, targetAngle);
});
```

**Визуальный результат:**
```
        ┌─────────────────┐
        │  ●              │  Технология 1 (r=50, θ=45°)
        │  ●              │  Технология 2 (r=150, θ=45°)
        │     ●           │  Технология 3 (r=250, θ=50°)
        │     ●           │  Технология 4 (r=100, θ=50°)
        └─────────────────┘
```

**Характеристики:**
- Фиксированное количество углов (например, 5-10 "лучей")
- Технологии распределяются по этим углам
- Радиус определяется готовностью (не изменяется)
- Технологии выстраиваются в "лучи" от центра

**Преимущества:**
- ✅ **Визуальная структура:** Явные "лучи" от центра, легко воспринимать
- ✅ **Группировка по готовности:** Технологии на одном "луче" легко сравнивать
- ✅ **Упорядоченность:** Визуально более структурированный вид
- ✅ **Легкость навигации:** Проще находить технологии в квадранте

**Недостатки:**
- ❌ **Ограниченное количество углов:** При большом количестве технологий возможны наложения
- ❌ **Потеря точности угла:** Угол не отражает уникальность технологии
- ❌ **Сложность при большом количестве:** Много технологий на одном "луче" может создать визуальный шум

### Подход 2: Гибридный подход (Группировка с небольшим разбросом)

**Принцип:** Технологии группируются по основным углам, но с небольшим случайным/детерминированным смещением для избежания точных наложений.

```javascript
// Псевдокод
const baseAngles = calculateBaseAngles(group.length, angleMin, angleMax);
group.forEach((t, index) => {
  const baseAngle = baseAngles[index % baseAngles.length];
  const offset = calculateSmallOffset(t.id, t.radius); // Небольшое смещение
  targetAngle = baseAngle + offset;
  newPos = polarToCartesian(radius, targetAngle);
});
```

**Визуальный результат:**
```
        ┌─────────────────┐
        │  ●              │  Технология 1 (r=50, θ=45.2°)
        │  ●              │  Технология 2 (r=150, θ=45.1°)
        │     ●           │  Технология 3 (r=250, θ=50.3°)
        │     ●           │  Технология 4 (r=100, θ=50.1°)
        └─────────────────┘
```

**Характеристики:**
- Основные углы определяют "лучи"
- Небольшое смещение (±0.5-2°) для каждой технологии
- Смещение зависит от ID и радиуса (детерминированно)
- Сохраняется визуальная структура "лучей"

**Преимущества:**
- ✅ **Структура + вариативность:** Сохраняется структура "лучей", но есть вариативность
- ✅ **Меньше наложений:** Небольшое смещение помогает избежать точных наложений
- ✅ **Визуальная группировка:** Технологии визуально группируются, но не накладываются точно
- ✅ **Гибкость:** Можно регулировать степень группировки

**Недостатки:**
- ❌ **Сложность реализации:** Требует балансировки между группировкой и разбросом
- ❌ **Параметры:** Нужно подобрать оптимальные параметры смещения

### Подход 3: Адаптивная группировка по радиусам

**Принцип:** Технологии группируются по углам, но количество "лучей" зависит от количества технологий и их распределения по радиусам.

```javascript
// Псевдокод
const radiusGroups = groupTechnologiesByRadius(group, radiusTolerance);
radiusGroups.forEach(radiusGroup => {
  const angles = calculateOptimalAngles(radiusGroup.length, angleMin, angleMax);
  radiusGroup.forEach((t, index) => {
    targetAngle = angles[index];
    newPos = polarToCartesian(t.radius, targetAngle);
  });
});
```

**Характеристики:**
- Технологии сначала группируются по радиусам (в пределах допуска)
- Для каждой группы радиусов рассчитываются оптимальные углы
- Технологии на близких радиусах могут иметь разные углы
- Технологии на разных радиусах могут иметь одинаковые углы

**Преимущества:**
- ✅ **Адаптивность:** Количество "лучей" адаптируется к количеству технологий
- ✅ **Оптимизация:** Оптимальное использование пространства
- ✅ **Группировка:** Сохраняется визуальная структура

**Недостатки:**
- ❌ **Сложность:** Более сложный алгоритм
- ❌ **Нестабильность:** При изменении данных структура может меняться

### Подход 4: Слоистая структура (Radial Layers)

**Принцип:** Технологии группируются в "слои" по радиусам, внутри каждого слоя равномерно распределяются по углу.

```javascript
// Псевдокод
const layers = createRadialLayers(group, layerCount);
layers.forEach((layer, layerIndex) => {
  const layerAngle = calculateLayerAngle(layerIndex, layerCount);
  layer.forEach((t, index) => {
    const angleOffset = calculateOffsetInLayer(index, layer.length);
    targetAngle = layerAngle + angleOffset;
    newPos = polarToCartesian(t.radius, targetAngle);
  });
});
```

**Характеристики:**
- Технологии группируются в радиальные "слои"
- Каждый слой имеет свой основной угол
- Внутри слоя технологии распределяются с небольшим смещением
- Создает структуру концентрических "лучей"

**Преимущества:**
- ✅ **Структурированность:** Явная слоистая структура
- ✅ **Группировка:** Технологии визуально группируются по готовности
- ✅ **Масштабируемость:** Работает при любом количестве технологий

**Недостатки:**
- ❌ **Сложность:** Требует определения количества слоев
- ❌ **Потеря точности радиуса:** Может потребоваться округление радиусов до слоев

## Рекомендации

### Рекомендация 1: Гибридный подход (Приоритет 1)

**Обоснование:**
- Сохраняет визуальную структуру "лучей" для лучшего восприятия
- Позволяет избежать точных наложений через небольшое смещение
- Балансирует между структурой и использованием пространства
- Относительно простая реализация

**Реализация:**
1. Определить оптимальное количество "лучей" (например, 5-8 для квадранта)
2. Распределить технологии по этим "лучам" (циклически или по готовности)
3. Добавить небольшое детерминированное смещение (±0.5-2°) на основе ID и радиуса
4. Сохранить радиус из математической модели

**Код:**
```javascript
function applyRayBasedDistribution(group, q) {
  const ANGLE_SPAN = 90 - (POSITION_ANGLE_PAD * 2);
  const angleMin = q.startAngle + POSITION_ANGLE_PAD;
  const angleMax = q.startAngle + POSITION_ANGLE_PAD + ANGLE_SPAN;

  // Определяем количество "лучей" (адаптивно)
  const optimalRayCount = Math.min(8, Math.max(3, Math.ceil(Math.sqrt(group.length))));
  const rayAngles = [];
  for (let i = 0; i < optimalRayCount; i++) {
    const ratio = i / (optimalRayCount - 1);
    rayAngles.push(angleMin + ratio * ANGLE_SPAN);
  }

  // Сортируем по радиусу для группировки по готовности
  group.sort((a, b) => {
    const polarA = cartesianToPolar(a.x, a.y);
    const polarB = cartesianToPolar(b.x, b.y);
    return polarA.radius - polarB.radius;
  });

  // Распределяем по "лучам" с небольшим смещением
  group.forEach((t, index) => {
    const rayIndex = index % optimalRayCount;
    const baseAngle = rayAngles[rayIndex];

    // Небольшое детерминированное смещение на основе ID и радиуса
    const polar = cartesianToPolar(t.x, t.y);
    const offset = ((t.id * 7 + Math.floor(polar.radius / 10)) % 5 - 2) * 0.3; // ±0.6°

    const targetAngle = baseAngle + offset;
    const newPos = polarToCartesian(polar.radius, targetAngle);
    t.x = newPos.x;
    t.y = newPos.y;
  });
}
```
