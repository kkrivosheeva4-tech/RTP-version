# Чек-лист сценариев для регрессионной проверки

**Версия:** 1.1
**Дата:** 04.03.2026
**Назначение:** Фиксация ключевых сценариев и шагов для ручной проверки после рефакторинга (и перед ним — как эталон поведения). Используется на этапе 0.3 плана (docs/FRONTEND_FINAL_PLAN.md).

---

## Как использовать

Перед началом этапа рефакторинга и после каждого этапа выполнить проверку по чек-листу. **После завершения всех этапов рефакторинга (0–7)** обязательно пройти полную проверку и указать дату и результат в блоке «Итог». Отмечать пункты по мере прохождения (✓). При падении сценария — зафиксировать шаги до воспроизведения и ожидаемый результат.

---

## 1. Загрузка радара

| # | Шаг | Ожидаемый результат | ✓ |
|---|-----|---------------------|---|
| 1.1 | Открыть главную страницу (index.html), выбрать предприятие и перейти на радар | Открывается страница радара (radar.html) |
| 1.2 | Дождаться окончания загрузки данных | Радар отображается: кольца, сектора, blip'ы технологий; нет бесконечного спиннера |
| 1.3 | Проверить консоль браузера | Нет критичных ошибок (красные сообщения), возможны предупреждения |
| 1.4 | Перезагрузить страницу радара с выбранным предприятием в localStorage | Радар загружается с тем же предприятием |

---

## 2. Фильтры

| # | Шаг | Ожидаемый результат | ✓ |
|---|-----|---------------------|---|
| 2.1 | Открыть панель фильтров (если есть кнопка/область) | Панель фильтров открывается |
| 2.2 | Выбрать «Функциональный блок» (block) | Список блоков отображается, выбор применяется |
| 2.3 | Выбрать «Функция» (function) | Список функций отображается, выбор применяется |
| 2.4 | Выбрать тип технологии (techType) и уровень (level) | Фильтры применяются, набор blip'ов на радаре обновляется |
| 2.5 | Сбросить фильтры (если есть кнопка сброса) | Все фильтры сбрасываются, отображаются все технологии |
| 2.6 | Ввести текст в поле поиска | Список технологий на радаре и в сайдбаре фильтруется по введённому тексту |

---

## 3. CRUD технологии (добавление / редактирование / удаление)

| # | Шаг | Ожидаемый результат | ✓ |
|---|-----|---------------------|---|
| 3.1 | Нажать «Добавить технологию» (или аналог) | Открывается модальное окно формы добавления |
| 3.2 | Заполнить обязательные поля и отправить форму | Технология добавляется, модалка закрывается, на радаре появляется новый blip (или обновляется список) |
| 3.3 | Кликнуть по blip технологии на радаре | Открывается панель деталей технологии |
| 3.4 | Нажать «Редактировать» в панели деталей | Открывается форма редактирования с заполненными полями |
| 3.5 | Изменить поле и сохранить | Изменения сохраняются, панель/модалка закрываются, радар обновляется |
| 3.6 | Нажать «Удалить» и подтвердить удаление | Технология удаляется, blip исчезает с радара, панель закрывается |

## 4. Экспорт

| # | Шаг | Ожидаемый результат | ✓ |
|---|-----|---------------------|---|
| 4.1 | Нажать кнопку экспорта в PDF (например «Экспорт отчёта» / Export PDF) | Открывается модальное окно выбора полей и фильтров экспорта |
| 4.2 | Выбрать поля для экспорта и при необходимости фильтры | Выбор сохраняется в UI |
| 4.3 | Запустить генерацию PDF | Отображается индикатор загрузки/генерации |
| 4.4 | Дождаться окончания | PDF скачивается или открывается; при ошибке показывается сообщение пользователю |

## 5. Вход (авторизация)

| # | Шаг | Ожидаемый результат | ✓ |
|---|-----|---------------------|---|
| 5.1 | Открыть страницу входа (auth.html) | Отображается форма входа |
| 5.2 | Ввести учётные данные тестового пользователя backend и войти | Выполняется вход через backend API, переход на главную или радар |
| 5.3 | На радаре проверить видимость кнопок (добавить/редактировать/удалить) в зависимости от роли | Для архитектора/админа кнопки видны; для гостя — скрыты или ограничены |

## 6. Админ-панель

| # | Шаг | Ожидаемый результат | ✓ |
|---|-----|---------------------|---|
| 6.1 | Открыть страницу админки (admin.html) | Загружается админ-панель (дашборд или первый раздел) |
| 6.2 | Переключиться на раздел «Пользователи» | Отображается таблица/список пользователей |
| 6.3 | Переключиться на раздел «Журнал аудита» | Отображаются записи аудита (при наличии данных) |

| 6.5 | Выполнить действие в разделе (например добавить/изменить пользователя, если доступно) | Действие выполняется, данные обновляются или показывается сообщение |
