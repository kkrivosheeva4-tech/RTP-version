# Матрица ролей приложения "Радар технологий"

## Описание

Данный документ описывает матрицу ролей и прав доступа для всех функций приложения "Радар технологий". В таблице указано, какие роли имеют доступ к выполнению конкретных функций.

## Роли пользователей

- **Гость (guest)** — пользователь с минимальными правами доступа
- **Архитектор (architect)** — пользователь с правами на редактирование и управление данными
- **Директор (director)** — пользователь с расширенными правами для принятия решений
- **Руководитель проекта (project_manager)** — пользователь с расширенными правами для управления проектами
- **Администратор (admin)** — пользователь с полными правами доступа ко всем функциям системы

## Условные обозначения

- ✅ — Доступ разрешен
- ❌ — Доступ запрещен
- ⚠️ — Ограниченный доступ (частичный функционал)

## Особенности ролей

### Гость (guest)
- Минимальный уровень доступа
- Может только просматривать радар технологий и информацию о технологиях
- Не может вносить изменения в данные
- Не имеет доступа к административным функциям

### Архитектор (architect)
- Имеет все права гостя
- Может добавлять, редактировать и удалять технологии
- Имеет доступ к административной панели
- Может управлять пользователями
- Может просматривать журнал аудита
- Может экспортировать данные в PDF
- Работает на единой странице радара (RMK-director.html) со всеми функциями

### Директор (director)
- Имеет все права архитектора
- Работает на единой странице радара (RMK-director.html)
- Может управлять вендорами и интеграторами
- Может прикреплять файлы к технологиям
- Видит модифицированный радар с позиционированием по готовности
- Не может управлять резервными копиями (только администратор)

### Руководитель проекта (project_manager)
- Имеет те же права, что и директор
- Работает на единой странице радара (RMK-director.html)
- Может управлять вендорами и интеграторами
- Может прикреплять файлы к технологиям
- Не может управлять резервными копиями (только администратор)

### Администратор (admin)
- Имеет все права архитектора
- Полный доступ ко всем функциям системы
- Может управлять резервными копиями (создание, восстановление, удаление)
- Может экспортировать все данные (пользователи, журнал аудита)
- Работает на единой странице радара (RMK-director.html)
- Полный контроль над системой

## Примечания

1. **Проверка прав доступа**: Все функции, требующие прав архитектора или выше, проверяются через функцию `checkArchitectRole()`, которая возвращает `true` для ролей: `architect`, `admin`, `director`, `project_manager`.

2. **Проверка прав директора**: Функции, специфичные для директорской страницы, проверяются через функцию `checkDirectorRole()`, которая возвращает `true` только для ролей: `director` и `project_manager`.

3. **Журналирование действий**: Все действия пользователей (добавление, редактирование, удаление) логируются в журнал аудита, доступный для просмотра архитекторам, директорам, РП и администраторам.

4. **Резервные копии**: Управление резервными копиями доступно только администратору для обеспечения безопасности данных.

5. **Экспорт данных**: Экспорт пользователей и журнала аудита доступен только администратору, так как содержит конфиденциальную информацию.

6. **Единая страница радара**: Все роли работают на единой странице `RMK-director.html`. Страница `RMK.html` используется только для редиректа на `RMK-director.html` (для обратной совместимости). Страница `RMK-director.html` использует отдельный файл данных `enterpriseData-director.json`, который содержит дополнительные поля (vendors, files).
