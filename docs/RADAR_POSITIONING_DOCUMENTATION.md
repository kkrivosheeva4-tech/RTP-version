## Обзор системы позиционирования

Система позиционирования технологий на радаре использует **полярную систему координат** с центром в точке (CENTER_X, CENTER_Y). Каждая технология позиционируется на основе двух параметров:

- **Угол (θ)** - определяется принадлежностью технологии к направлению цифрового развития (квадранту)
- **Радиус (r)** - вычисляется через математическую модель на основе факторов готовности

### Основные принципы

1. **Квадранты определяются через направления**: Технология размещается в квадранте на основе её направлений цифрового развития (маппинг `directionToQuadrant`)

2. **Радиус зависит от готовности**: Чем выше готовность технологии к внедрению, тем ближе она к центру радара (меньше радиус)

3. **Разведение при наложениях**: Алгоритм автоматически разводит технологии, которые накладываются друг на друга

## Математическая модель позиционирования

### Общая структура модели

Математическая модель использует **логистическую функцию** для преобразования факторов готовности в позицию на радаре.

#### Конвейер вычисления радиуса

```
1. Факторы готовности (s_ik) → 2. Нормализация (x_ik) → 3. Сводный показатель (z_i) →
4. Логистическая функция (p_i) → 5. Радиус (r_i)
```

### Детальное описание этапов

#### 1. Факторы готовности (s_ik)

Модель использует четыре фактора:

| Фактор | Диапазон | Описание |
|--------|----------|----------|
| `techRead` | 0-3 | Технологическая готовность (среднее по выбранным предприятиям) |
| `organRead` | 0-3 | Организационная готовность (среднее по выбранным предприятиям) |
| `funcCover` | 0-3 | Покрытие функций (общее значение для технологии) |
| `trlStage` | 1-3 | TRL стадия (Technology Readiness Level) |

**Особенности расчета techRead и organRead:**
- Если выбраны предприятия в фильтре, вычисляется среднее значение только по выбранным предприятиям
- Если фильтр не применен, используются все предприятия технологии
- Если у технологии нет данных по предприятиям, используются общие значения (fallback)

#### 2. Нормализация факторов (x_ik)

Все факторы нормализуются в диапазон [0, 1]:

```javascript
x_techRead = techRead / 3        // 0-3 → 0-1
x_organRead = organRead / 3      // 0-3 → 0-1
x_funcCover = funcCover / 3      // 0-3 → 0-1
x_trlStage = (trlStage - 1) / 2  // 1-3 → 0-1
```

**Обработка отсутствующих данных:**
- Если `techRead` или `organRead` отсутствуют (null/undefined), используется среднее значение 0.5 вместо 0
- Это позволяет различать "отсутствие данных" и "минимальная готовность"

#### 3. Сводный показатель (z_i)

Вычисляется как взвешенная сумма нормализованных факторов плюс смещение (bias):

```javascript
z_i = w_techRead * x_techRead +
      w_organRead * x_organRead +
      w_funcCover * x_funcCover +
      w_trlStage * x_trlStage +
      bias
```

**Веса факторов (по умолчанию):**
- `w_techRead = 0.30` (30%)
- `w_organRead = 0.30` (30%)
- `w_funcCover = 0.20` (20%)
- `w_trlStage = 0.20` (20%)
- **Сумма весов = 1.0** (автоматическая нормализация при переопределении)

**Смещение (bias):**
- По умолчанию: `bias = -0.6`
- Калибровка модели:
  - Максимальные параметры (все = 3) → r ≈ 17% (близко к центру)
  - Средние параметры (все = 1.5) → r ≈ 60% (среднее положение)
  - Низкие параметры (все = 0) → r ≈ 92% (у края)

**Конфигурируемость:**
Параметры модели можно переопределить через `window.RadarModelConfig`:
```javascript
window.RadarModelConfig = {
  alpha: 4,              // Параметр чувствительности логистической функции
  bias: -0.6,            // Смещение для калибровки
  weights: {
    techRead: 0.30,
    organRead: 0.30,
    funcCover: 0.20,
    trlStage: 0.20
  }
};
```

#### 4. Логистическая функция (p_i)

Преобразует сводный показатель в вероятность готовности:

```javascript
p_i = 1 / (1 + exp(-α * z_i))
```

где:
- `α` (alpha) = 4 (по умолчанию) - параметр чувствительности
- `p_i ∈ (0, 1)` - степень близости к внедрению (вероятность готовности)

**Свойства логистической функции:**
- При `z_i → +∞`: `p_i → 1` (высокая готовность)
- При `z_i → -∞`: `p_i → 0` (низкая готовность)
- При `z_i = 0`: `p_i = 0.5` (средняя готовность)

#### 5. Вычисление радиуса (r_i)

Радиус вычисляется как инверсия вероятности готовности:

```javascript
r_i = 100 * (1 - p_i)
```

**Интерпретация:**
- `p_i = 1` (максимальная готовность) → `r_i = 0` (центр радара)
- `p_i = 0` (минимальная готовность) → `r_i = 100` (край радара)
- `p_i = 0.5` (средняя готовность) → `r_i = 50` (середина)

**Гарантия диапазона:**
Радиус гарантированно находится в диапазоне `(0, 100)`:
- Если `r_i ≤ 0`: устанавливается `r_i = 0.01` (EPSILON)
- Если `r_i ≥ 100`: устанавливается `r_i = 99.99`

### Вычисление угла (θ)

Угол определяется принадлежностью технологии к квадранту:

1. **Определение квадранта**: По первому направлению технологии через маппинг `directionToQuadrant`

2. **Базовый угол квадранта**: Каждый квадрант имеет `startAngle` (например, квадрант 1: 0°, квадрант 2: 90°, квадрант 3: 180°, квадрант 4: 270°)

3. **Распределение внутри квадранта**: Используется золотой угол (Golden Angle = 137.50776405003785°) для равномерного распределения:

```javascript
ANGLE_PAD = 8°  // Отступ от границ квадранта
ANGLE_SPAN = 90 - (ANGLE_PAD * 2) = 74°  // Доступный диапазон
angleOffset = (tech.id * GOLDEN_ANGLE) % ANGLE_SPAN
theta = startAngle + ANGLE_PAD + angleOffset
```

### Преобразование в декартовы координаты

После вычисления полярных координат (θ, r), они преобразуются в декартовы:

```javascript
// Масштабирование радиуса из процентов (0-100) в пиксели
maxR = количество_колец * RADIUS_STEP
availableRadius = maxR - POSITION_PAD
radius_pixels = POSITION_PAD + (r_i / 100) * availableRadius

// Преобразование полярных координат в декартовы
x = CENTER_X + radius_pixels * cos(θ)
y = CENTER_Y + radius_pixels * sin(θ)
```

**Константы:**
- `CENTER_X = 500` (пикселей)
- `CENTER_Y = 500` (пикселей)
- `RADIUS_STEP = 140` (пикселей между кольцами)
- `POSITION_PAD = 30` (пикселей отступ от границ)

## Расчет оценки "Покрытие функций"

Оценка "Покрытие функций" (`funcCover`) вычисляется на основе процентного покрытия функциональных блоков технологии.

### Алгоритм расчета

#### 1. Определение покрытых функций

Технология имеет массив `functionCoverage` - список функций, которые она покрывает.

#### 2. Определение блоков технологии

Технология имеет массив `blocks` - список функциональных блоков, к которым она относится.

#### 3. Подсчет функций в блоках

Для каждого блока технологии определяется общее количество функций через маппинг `functionToBlock.json`:

```javascript
// Пример структуры functionToBlock.json
{
  "Функция 1": [1, 2],      // Функция относится к блокам 1 и 2
  "Функция 2": [1],
  "Функция 3": [2, 3]
}

// Подсчет функций в блоке 1: 2 функции
// Подсчет функций в блоке 2: 2 функции
// Подсчет функций в блоке 3: 1 функция
```

#### 4. Учет важности функций (опционально)

Если включен учет важности (`useWeights: true`), используется взвешенное покрытие:

```javascript
// Загрузка весов из functionWeights.json
weights = {
  "Функция 1": 1.5,  // Важная функция
  "Функция 2": 1.0,  // Обычная функция
  "Функция 3": 0.8   // Менее важная функция
}

// Взвешенная сумма всех функций в блоках
totalWeightedFunctions = Σ(weight[func] для всех func в блоках)

// Взвешенная сумма покрытых функций
coveredWeightedSum = Σ(weight[func] для func в functionCoverage)
```

Если веса не используются или отсутствуют, все функции имеют вес = 1.0.

#### 5. Вычисление процента покрытия

```javascript
coveragePercent = coveredWeightedSum / totalWeightedFunctions
```

**Пример:**
- Технология относится к блоку 1 (2 функции) и блоку 2 (2 функции)
- Всего функций в блоках: 4
- Покрыто функций: 2
- Процент покрытия: 2/4 = 0.5 (50%)

#### 6. Преобразование процента в оценку funcCover (0-3)

Используется функция `convertCoveragePercentToFuncCover`:

```javascript
function convertCoveragePercentToFuncCover(coveragePercent) {
  if (coveragePercent === 0) return 0;
  if (coveragePercent >= 1.0) return 3;

  const rawValue = coveragePercent * 3;
  const threshold = 0.05; // 5% порог для плавных переходов

  if (rawValue < 1 + threshold) return 1;
  if (rawValue < 2 - threshold) return Math.round(rawValue);
  if (rawValue < 2 + threshold) return 2;
  if (rawValue < 3 - threshold) return Math.round(rawValue);
  return 3;
}
```

**Маппинг процента в оценку:**

| Процент покрытия | funcCover | Описание |
|------------------|-----------|----------|
| 0% | 0 | Не покрывает необходимые функции |
| 1-33% | 1 | Низкое покрытие функций |
| 34-67% | 2 | Среднее покрытие функций |
| 68-100% | 3 | Высокое покрытие функций |

**Особенности:**
- Если есть хотя бы одна функция, `funcCover` минимум = 1
- Используется плавное округление с порогом 5% для избежания резких переходов

### Fallback логика

Если блоки не указаны или данные не загружены, используется legacy расчет:

```javascript
function calculateFuncCoverLegacy(funcCount) {
  if (funcCount === 0) return 0;
  if (funcCount === 1) return 1;
  if (funcCount >= 2 && funcCount <= 3) return 2;
  return 3; // 4+ функций
}
```

### Синхронный расчет

Для использования в позиционировании (которое должно быть синхронным) используется метод `calculateFuncCoverSync`, который работает с предзагруженными данными из localStorage.

## Факторы, влияющие на размер фигуры

Размер фигуры технологии на радаре зависит от **количества вендоров** (только на директорской странице).

### Логика определения размера

```javascript
function calculateElementSize(tech) {
  const isDirectorPage = document.body.id === 'rmk-director';

  if (isDirectorPage) {
    const vendorCount = tech.vendors ? tech.vendors.length : 0;

    if (vendorCount <= 1) {
      size = 8;   // Маленький размер (1 вендор или нет вендоров)
    } else if (vendorCount === 2 || vendorCount === 3) {
      size = 14;  // Средний размер (2-3 вендора)
    } else {
      size = 20;  // Большой размер (4+ вендора)
    }
  } else {
    size = 10;    // Стандартный размер (не директорская страница)
  }

  return size; // Радиус фигуры в пикселях
}
```

### Таблица размеров

| Количество вендоров | Размер (радиус) | Применение |
|---------------------|-----------------|------------|
| 0-1 | 8px | Маленькая фигура |
| 2-3 | 14px | Средняя фигура |
| 4+ | 20px | Большая фигура |
| Не директорская страница | 10px | Стандартный размер |

### Влияние размера на позиционирование

Размер фигуры учитывается при:

1. **Расчете углового размера**: Для корректировки границ квадранта
   ```javascript
   angularSize = arcsin(elementSize * 1.2 / radius) * 180 / π
   ```

2. **Минимальном расстоянии между технологиями**:
   ```javascript
   minDistance = sizeA + sizeB + 4  // Сумма радиусов + зазор
   ```

3. **Алгоритме разведения**: Технологии с большими размерами требуют больше места

## Функция контроля наложений технологий

Функция `applyNonOverlappingLayout` предотвращает наложение технологий друг на друга через итеративный алгоритм разведения.

### Общая структура алгоритма

```
1. Группировка по квадрантам
2. Равномерное распределение по углу
3. Ограничение в пределах квадранта
4. Предварительное разведение идентичных координат
5. Итеративное разведение близких точек
6. Финальное разведение по радиусу (если необходимо)
```

### Детальное описание этапов

#### 1. Группировка по квадрантам

Технологии группируются по квадрантам для независимой обработки:

```javascript
groups = new Map()
renderData.forEach(tech => {
  const key = `${tech.quadrant}`
  if (!groups.has(key)) groups.set(key, [])
  groups.get(key).push(tech)
})
```

#### 2. Равномерное распределение по углу

Для каждой группы применяется функция `applyUniformAngleDistribution`:

**Принцип работы:**
- Технологии с одинаковым радиусом (в пределах допуска 5px) распределяются равномерно по углу
- Технологии с разным радиусом сохраняют математически рассчитанную позицию

```javascript
// Группировка по радиусу
RADIUS_TOLERANCE = 5px
radiusGroups = группировать по Math.floor(radius / RADIUS_TOLERANCE)

// Для технологий с одинаковым радиусом - равномерное распределение
// Для технологий с уникальным радиусом - сохранение математической позиции
```

#### 3. Ограничение в пределах квадранта

Функция `clampToSectorRing` гарантирует, что технология не выходит за границы квадранта:

```javascript
function clampToSectorRing(tech) {
  // Границы квадранта с учетом размера элемента
  angleMin = startAngle + ANGLE_PAD + angularSize
  angleMax = startAngle + 90 - ANGLE_PAD - angularSize

  // Ограничение радиуса
  radius = clamp(radius, rMin, rMax)

  // Ограничение угла
  angle = clamp(angle, angleMin, angleMax)

  // Специальная обработка квадранта 4 (пересекает 0°/360°)
  if (quadrant === 4) {
    if (angle < 90) angle += 360
    // ... нормализация
  }
}
```

#### 4. Предварительное разведение идентичных координат

Если две технологии имеют практически идентичные координаты (разница < 0.5px), они разводятся по углу:

```javascript
COORDINATE_TOLERANCE = 0.5px

if (Math.abs(xA - xB) < COORDINATE_TOLERANCE &&
    Math.abs(yA - yB) < COORDINATE_TOLERANCE) {
  // Разведение по углу с учетом размеров элементов
  minAngularDistance = angularSizeA + angularSizeB + 2°
  // Применение смещения на основе ID технологии
}
```

#### 5. Итеративное разведение близких точек

Основной алгоритм разведения использует **метод отталкивания**:

```javascript
MAX_ITER = 80-150 (адаптивно в зависимости от количества технологий)
CONVERGENCE_THRESHOLD = 0.1px
DAMPING_DECAY = 0.98

for (iter = 0; iter < MAX_ITER; iter++) {
  dampingFactor *= DAMPING_DECAY  // Затухание для стабильности

  for (каждую пару технологий (A, B)) {
    distance = sqrt((xB - xA)² + (yB - yA)²)
    minDistance = sizeA + sizeB + 4  // Сумма радиусов + зазор

    if (distance < minDistance) {
      overlap = minDistance - distance
      forceMultiplier = (distance < minDistance * 0.5) ? 1.5 : 1.0

      // Вектор отталкивания
      shiftX = (dx / distance) * (overlap / 2) * forceMultiplier * dampingFactor
      shiftY = (dy / distance) * (overlap / 2) * forceMultiplier * dampingFactor

      // Применение смещения
      A.x -= shiftX
      A.y -= shiftY
      B.x += shiftX
      B.y += shiftY

      // Ограничение в пределах квадранта
      clampToSectorRing(A)
      clampToSectorRing(B)
    }
  }

  // Проверка сходимости
  if (maxMovement < CONVERGENCE_THRESHOLD) break
}
```

**Особенности алгоритма:**
- **Адаптивное минимальное расстояние**: Зависит от количества технологий в квадранте
  ```javascript
  if (techCount > 10) minDistance *= 1.3
  else if (techCount > 5) minDistance *= 1.15
  ```
- **Учет размеров элементов**: Минимальное расстояние = сумма радиусов + зазор 4px
- **Коэффициент затухания**: Уменьшается с каждой итерацией для стабильности
- **Усиленное отталкивание**: Для очень близких точек (distance < 0.5 * minDistance) применяется множитель 1.5

#### 6. Финальное разведение по радиусу

Если после всех итераций остались наложения, применяется радикальное разведение по радиусу:

```javascript
for (каждую пару технологий (A, B)) {
  if (все еще накладываются) {
    radiusDiff = minDistance * 1.2
    // Одна технология ближе к центру, другая дальше
    newRadiusA = max(rMin, radiusA - radiusDiff / 2)
    newRadiusB = min(rMax, radiusB + radiusDiff / 2)

    // Обновление позиций с сохранением угла
    newPosA = polarToCartesian(newRadiusA, angleA)
    newPosB = polarToCartesian(newRadiusB, angleB)
  }
}
```

### Оптимизация для больших наборов данных

Если в квадранте более 20 технологий и доступен модуль `SpatialIndex`, используется оптимизированный алгоритм:

```javascript
if (group.length > 20 && window.SpatialIndex) {
  window.SpatialIndex.optimizeLayoutWithSpatialIndex(group, bounds, {
    maxIterations: 80,
    dampingFactor: 0.98,
    convergenceThreshold: 0.1
  })
}
```

### Параметры алгоритма

| Параметр | Значение | Описание |
|----------|----------|-----------|
| `MIN_BLIP_DISTANCE` | 28px | Базовое минимальное расстояние |
| `MAX_ITER` | 80-150 | Максимальное количество итераций (адаптивно) |
| `CONVERGENCE_THRESHOLD` | 0.1px | Порог сходимости |
| `DAMPING_DECAY` | 0.98 | Коэффициент затухания |

## Технические детали реализации

### Кеширование позиций

Для оптимизации производительности используется кеш позиций:

```javascript
positionCache = new Map()
cacheKey = `${version}:${tech.id}:${techRead}:${organRead}:${funcCover}:${trlStage}:${directions}`

// Проверка кеша перед расчетом
if (positionCache.has(cacheKey)) {
  return positionCache.get(cacheKey)
}

// Сохранение результата
positionCache.set(cacheKey, { x, y })
```

**Ограничение размера кеша:** Максимум 1000 записей (удаление старых при превышении)

### Валидация входных данных

Все входные значения факторов валидируются:

```javascript
function validateAndNormalizeFactor(value, factorName, min, max, defaultValue) {
  if (value === null || value === undefined) return null
  const numValue = Number(value)
  if (isNaN(numValue)) return defaultValue
  return Math.max(min, Math.min(max, numValue))  // Ограничение диапазона
}
```

### Обработка отсутствующих данных

1. **Предсказание через k-NN**: Если доступен модуль `MissingDataPredictor`, отсутствующие значения предсказываются на основе похожих технологий

2. **Использование средних значений**: Вместо 0 используется 0.5 для интерполяции

3. **Логирование**: Все случаи отсутствующих данных логируются для отладки

### Тестирование калибровки

Доступны функции для тестирования модели:

```javascript
// Тест конкретных параметров
Positioning.testCalibration({
  techRead: 3,

// Тест всех сценариев
```

### Зависимости модулей

Позиционирование зависит от следующих модулей:

- `radar-utils.js`: Функции преобразования координат (`polarToCartesian`, `cartesianToPolar`)
- `func-cover-utils.js`: Расчет покрытия функций

## Заключение

Система позиционирования технологий на радаре представляет собой сложную математическую модель, которая:

1. **Учитывает множественные факторы готовности** через взвешенную логистическую функцию
2. **Автоматически разводит технологии** при наложениях с сохранением математической точности
3. **Адаптируется к различным сценариям** через конфигурируемые параметры
4. **Оптимизирована для производительности** через кеширование и пространственные индексы

Модель обеспечивает визуальное представление готовности технологий к внедрению, где близость к центру радара соответствует высокой готовности, а удаленность - низкой.
