## Введение

Оценка **"Покрытие функций"** (`funcCover`) — это числовая метрика в диапазоне от 0 до 3, которая показывает, насколько полно технология покрывает необходимые функции в рамках функциональных блоков, к которым она относится.

### Назначение

- Оценка полноты функционального покрытия технологии
- Сравнение технологий по степени покрытия функций
- Использование в расчете позиционирования технологий на радаре

### Диапазон значений

| Значение | Описание | Процент покрытия |
|----------|----------|------------------|
| 0 | Не покрывает необходимые функции | 0% |
| 1 | Низкое покрытие функций | 1-33% |
| 2 | Среднее покрытие функций | 34-67% |
| 3 | Высокое покрытие функций | 68-100% |

---

## Обзор алгоритма

Расчет оценки "Покрытие функций" выполняется в несколько этапов:

1. **Определение покрытых функций** — из массива `functionCoverage` технологии
2. **Определение функциональных блоков** — из массива `blocks` технологии
3. **Подсчет общего количества функций** — во всех блоках технологии
4. **Учет важности функций** (опционально) — взвешенное покрытие
5. **Вычисление процента покрытия** — отношение покрытых функций к общему количеству
6. **Преобразование в оценку** — маппинг процента на шкалу 0-3

## Детальное описание расчета

### Шаг 1: Определение покрытых функций

Технология содержит массив `functionCoverage` — список функций, которые она покрывает.

**Пример:**
```json
{
  "id": 1,
  "name": "Технология анализа данных",
  "functionCoverage": [
    "Аналитика",
    "Разработка",
    "Тестирование и аналитика"
  ]
}
```

В данном примере технология покрывает 3 функции.

**Важно:** При расчете учитываются только те функции из `functionCoverage`, которые действительно относятся к блокам технологии. Функции, не относящиеся к указанным блокам, игнорируются.

### Шаг 2: Определение функциональных блоков

Технология содержит поле `block` (один блок) или `blocks` (массив блоков), к которым она относится.

**Пример:**
```json
{
  "id": 1,
  "block": 18,
  "blocks": [18, 19]  // если технология относится к нескольким блокам
}
```

### Шаг 3: Подсчет общего количества функций в блоках

Для определения общего количества функций используется файл `functionToBlock.json`, который содержит маппинг функций на блоки.

**Структура `functionToBlock.json`:**
```json
{
  "Геологоразведка": 24,
  "Моделирование месторождений": 24,
  "Лабораторные исследования": [1, 2, 3],
  "Маркшейдерские работы": 1,
  "Буровые работы": 1
}
```

**Алгоритм подсчета:**

1. Для каждого блока технологии определяется список всех функций, которые к нему относятся
2. Если функция относится к нескольким блокам (массив в `functionToBlock.json`), она учитывается в каждом из них
3. Подсчитывается общее количество уникальных функций во всех блоках технологии

**Пример:**
- Технология относится к блоку 1
- В блоке 1 находятся функции: "Маркшейдерские работы", "Буровые работы", "Лабораторные исследования"
- Всего функций в блоке 1: 3

### Шаг 4: Учет важности функций (опционально)

Если включен учет важности функций (`useWeights: true`), используется взвешенное покрытие.

**Файл `functionWeights.json` (опциональный):**
```json
{
  "Аналитика": 1.5,
  "Разработка": 1.0,
  "Тестирование и аналитика": 0.8
}
```

**Расчет взвешенной суммы:**

1. **Взвешенная сумма всех функций в блоках:**
   ```
   totalWeightedFunctions = Σ(weight[func] для всех func в блоках технологии)
   ```

2. **Взвешенная сумма покрытых функций:**
   ```
   coveredWeightedSum = Σ(weight[func] для func в functionCoverage)
   ```

**Если веса не используются:**
- Все функции имеют вес = 1.0
- Взвешенная сумма равна простому количеству функций

### Шаг 5: Фильтрация покрытых функций

**ИСПРАВЛЕНО:** Перед расчетом процента покрытия выполняется фильтрация функций из `functionCoverage`:

1. Проверяется, существует ли функция в маппинге `functionToBlock.json`
2. Проверяется, относится ли функция хотя бы к одному из блоков технологии
3. Учитываются только валидные функции, относящиеся к блокам технологии

**Пример:**
- Технология относится к блоку 5
- Технология покрывает функции: ["ТОиР", "Аналитика"]
- "ТОиР" относится к блоку 5 — учитывается
- "Аналитика" относится к блоку 19, а не к блоку 5 — не учитывается
- В расчете используется только "ТОиР"

### Шаг 6: Вычисление процента покрытия

Процент покрытия вычисляется как отношение валидных покрытых функций к общему количеству функций:

**Без учета весов:**
```
coveragePercent = validCoveredFunctions.length / totalFunctionsInBlocks
```

**С учетом весов:**
```
coveragePercent = coveredWeightedSum / totalWeightedFunctions
```

где `coveredWeightedSum` рассчитывается только для валидных функций.

**Результат:** число от 0 до 1 (0% до 100%)

### Шаг 7: Преобразование процента в оценку funcCover (0-3)

Используется функция `convertCoveragePercentToFuncCover`:

```javascript
function convertCoveragePercentToFuncCover(coveragePercent) {
  // 0% покрытия = 0
  if (coveragePercent === 0) return 0;

  // 100% покрытия = 3
  if (coveragePercent >= 1.0) return 3;

  // Плавное преобразование с учетом порога 5%
  const rawValue = coveragePercent * 3;
  const threshold = 0.05;

  if (rawValue < 1 + threshold) return 1;
  if (rawValue < 2 - threshold) return Math.round(rawValue);
  if (rawValue < 2 + threshold) return 2;
  if (rawValue < 3 - threshold) return Math.round(rawValue);
  return 3;
}
```

**Особенности преобразования:**

- Если есть хотя бы одна функция, `funcCover` минимум = 1
- Используется плавное округление с порогом 5% для избежания резких переходов между категориями
- Гарантируется диапазон [0, 3]

**Маппинг процента в оценку:**

| Процент покрытия | rawValue (coveragePercent × 3) | funcCover | Описание |
|------------------|--------------------------------|-----------|----------|
| 0% | 0.0 | 0 | Не покрывает необходимые функции |
| 1-33% | 0.03-0.99 | 1 | Низкое покрытие функций |
| 34-67% | 1.02-2.01 | 2 | Среднее покрытие функций |
| 68-100% | 2.04-3.00 | 3 | Высокое покрытие функций |

## Примеры расчетов

### Пример 1: Простой расчет без весов

**Исходные данные:**
- Технология относится к блоку 1
- В блоке 1 находится 5 функций: ["Функция A", "Функция B", "Функция C", "Функция D", "Функция E"]
- Технология покрывает 2 функции: ["Функция A", "Функция B"]

**Расчет:**
1. Фильтрация: обе функции относятся к блоку 1 — обе валидны
2. Покрыто функций: 2
3. Всего функций в блоке: 5
4. Процент покрытия: 2/5 = 0.4 (40%)
5. rawValue = 0.4 × 3 = 1.2
6. funcCover = 1 (так как 1.2 < 1 + 0.05)

**Результат:** `funcCover = 1` (Низкое покрытие функций)

### Пример 1a: Расчет с фильтрацией функций

**Исходные данные:**
- Технология относится к блоку 5
- В блоке 5 находится 1 функция: ["ТОиР"]
- Технология покрывает функции: ["ТОиР", "Аналитика"] (вторая функция не относится к блоку 5)

**Расчет:**
1. Фильтрация: "ТОиР" относится к блоку 5 — валидна; "Аналитика" относится к блоку 19 — не учитывается
2. Валидных покрытых функций: 1
3. Всего функций в блоке: 1
4. Процент покрытия: 1/1 = 1.0 (100%)
5. funcCover = 3

**Результат:** `funcCover = 3` (Высокое покрытие функций)

### Пример 2: Расчет с несколькими блоками

**Исходные данные:**
- Технология относится к блокам [1, 2]
- Блок 1 содержит 3 функции: ["Функция A", "Функция B", "Функция C"]
- Блок 2 содержит 4 функции: ["Функция D", "Функция E", "Функция F", "Функция G"]
- Технология покрывает 4 функции: ["Функция A", "Функция B", "Функция D", "Функция E"]

**Расчет:**
1. Фильтрация: все 4 функции относятся к блокам 1 или 2 — все валидны
2. Покрыто функций: 4
3. Всего функций в блоках: 3 + 4 = 7
4. Процент покрытия: 4/7 ≈ 0.571 (57.1%)
5. rawValue = 0.571 × 3 ≈ 1.713
6. funcCover = 2 (так как 1.713 находится в диапазоне [1.05, 1.95])

**Результат:** `funcCover = 2` (Среднее покрытие функций)

### Пример 3: Расчет с учетом весов

**Исходные данные:**
- Технология относится к блоку 1
- В блоке 1 находятся функции с весами:
  - "Функция A": вес 1.5
  - "Функция B": вес 1.0
  - "Функция C": вес 0.8
  - "Функция D": вес 1.2
- Технология покрывает: ["Функция A", "Функция B"]

**Расчет:**
1. Взвешенная сумма всех функций: 1.5 + 1.0 + 0.8 + 1.2 = 4.5
2. Взвешенная сумма покрытых функций: 1.5 + 1.0 = 2.5
3. Процент покрытия: 2.5/4.5 ≈ 0.556 (55.6%)
4. rawValue = 0.556 × 3 ≈ 1.668
5. funcCover = 2

**Результат:** `funcCover = 2` (Среднее покрытие функций)

### Пример 4: Полное покрытие

**Исходные данные:**
- Технология относится к блоку 1
- В блоке 1 находится 3 функции
- Технология покрывает все 3 функции

**Расчет:**
1. Процент покрытия: 3/3 = 1.0 (100%)
2. funcCover = 3

**Результат:** `funcCover = 3` (Высокое покрытие функций)

### Пример 5: Отсутствие покрытия

**Исходные данные:**
- Технология относится к блоку 1
- В блоке 1 находится 5 функций
- Технология не покрывает ни одной функции (`functionCoverage` пуст или отсутствует)

**Расчет:**
1. Процент покрытия: 0/5 = 0.0 (0%)
2. funcCover = 0

**Результат:** `funcCover = 0` (Не покрывает необходимые функции)

## Особенности и ограничения

### Особенности алгоритма

1. **Фильтрация покрытых функций (ИСПРАВЛЕНО):**
   - Учитываются только функции из `functionCoverage`, которые действительно относятся к блокам технологии
   - Функции, не относящиеся к указанным блокам, игнорируются
   - Это предотвращает завышение оценки при указании функций из других блоков

2. **Минимальное значение при наличии функций:**
   - Если технология покрывает хотя бы одну валидную функцию, `funcCover` не может быть меньше 1
   - Это гарантирует, что технологии с минимальным покрытием не получат оценку 0

3. **Плавные переходы:**
   - Используется порог 5% для плавного перехода между категориями
   - Это позволяет избежать резких скачков при небольших изменениях покрытия

4. **Обработка нескольких блоков:**
   - При расчете учитываются все функции из всех блоков технологии
   - Функции, относящиеся к нескольким блокам, учитываются в каждом блоке

5. **Взвешенное покрытие:**
   - По умолчанию учет весов включен (`useWeights: true`)
   - Если файл `functionWeights.json` отсутствует, все функции имеют вес 1.0

### Ограничения

1. **Зависимость от данных:**
   - Расчет зависит от наличия и корректности файлов `functionToBlock.json` и `functionWeights.json`
   - При отсутствии данных используется legacy-логика (простое подсчет количества функций)

2. **Legacy-логика (fallback):**
   Если блоки не указаны или данные не загружены, используется упрощенный расчет:
   ```javascript
   function calculateFuncCoverLegacy(funcCount) {
     if (funcCount === 0) return 0;
     if (funcCount === 1) return 1;
     if (funcCount >= 2 && funcCount <= 3) return 2;
     return 3; // 4+ функций
   }
   ```

3. **Синхронность расчетов:**
   - Для синхронных расчетов используется метод `calculateFuncCoverSync`
   - Требуется предварительная загрузка данных в кеш

## Используемые данные

### Входные данные технологии

| Поле | Тип | Описание | Обязательное |
|------|-----|----------|--------------|
| `functionCoverage` | `string[]` | Массив функций, которые покрывает технология | Да |
| `block` | `number` | ID функционального блока (один блок) | Нет* |
| `blocks` | `number[]` | Массив ID функциональных блоков | Нет* |

*Хотя бы одно из полей `block` или `blocks` должно быть указано для корректного расчета

### Внешние файлы данных

#### `functionToBlock.json`

Маппинг функций на функциональные блоки.

**Структура:**
```json
{
  "Название функции": number | number[]
}
```

**Пример:**
```json
{
  "Геологоразведка": 24,
  "Моделирование месторождений": 24,
  "Лабораторные исследования": [1, 2, 3]
}
```

**Расположение:** `/src/data/ru/functionToBlock.json`

#### `functionWeights.json` (опциональный)

Веса важности функций для взвешенного расчета.

**Структура:**
```json
{
  "Название функции": number
}
```

**Пример:**
```json
{
  "Аналитика": 1.5,
  "Разработка": 1.0,
  "Тестирование": 0.8
}
```

**Расположение:** `/src/data/ru/functionWeights.json`

**Примечание:** Если файл отсутствует, все функции имеют вес 1.0

## Техническая реализация

### Модуль расчета

Расчет выполняется модулем `FuncCoverUtils`, расположенным в файле:
```
/src/js/modules/utils/func-cover-utils.js
```

### Основные функции

#### `calculateFuncCover(coveredFunctions, blockIds, options)`

Асинхронный расчет покрытия функций.

**Параметры:**
- `coveredFunctions` (string[]) — массив покрытых функций
- `blockIds` (number[]) — массив ID блоков технологии
- `options` (object) — опции расчета:
  - `useWeights` (boolean) — использовать ли веса функций (по умолчанию `true`)

**Возвращает:** `Promise<number>` — значение `funcCover` (0-3)

#### `calculateFuncCoverSync(coveredFunctions, blockIds, options)`

Синхронный расчет покрытия функций (требует предзагруженных данных).

**Параметры:** аналогично `calculateFuncCover`

**Возвращает:** `number` — значение `funcCover` (0-3)

#### `convertCoveragePercentToFuncCover(coveragePercent)`

Преобразование процента покрытия в оценку 0-3.

**Параметры:**
- `coveragePercent` (number) — процент покрытия (0-1)

**Возвращает:** `number` — значение `funcCover` (0-3)

### Инициализация

Модуль автоматически инициализируется при загрузке приложения:

```javascript
await window.FuncCoverUtils.init();
```

Инициализация включает:
- Загрузку данных `functionToBlock.json`
- Подсчет количества функций в каждом блоке
- Загрузку весов функций из `functionWeights.json` (если доступен)
- Кеширование данных в `localStorage` для офлайн-работы

### Кеширование

Модуль использует кеширование в `localStorage` для:
- Данных `functionToBlock.json` (ключ: `rtp_functionToBlock_data`)
- Версии данных (ключ: `rtp_functionToBlock_version`)
- Весов функций (ключ: `rtp_functionWeights`)

Это позволяет работать офлайн и ускоряет последующие загрузки.

## Заключение

Оценка "Покрытие функций" является важной метрикой для оценки полноты функционального покрытия технологии. Алгоритм расчета учитывает:

- Количество покрытых функций
- Общее количество функций в блоках технологии
- Важность функций (при наличии весов)
- Плавные переходы между категориями оценки

Расчет выполняется автоматически при сохранении технологии и используется в позиционировании технологий на радаре.
