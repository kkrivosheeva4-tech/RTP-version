5. Оптимизация работы с данными: заменить линейный поиск в горячих путях на `Map`/`Set`; рассмотреть отложенную обработку больших списков.

## Шаг 2 — инвентаризация горячих точек (наблюдения)

- Рендер радара: `renderRadar` полностью пересобирает blip'ы при каждом вызове `updateRadar`; тяжёлая точка при больших списках и частых фильтрах/поиске.
- Разводка точек: `applyNonOverlappingLayout` — O(n²) на группу (quadrant|ring) с многократными пересчётами координат; кандидат на оптимизацию/ранний выход.
- DOM генерация blip'ов: `createBlip` объявляет `starPath` внутри и создаёт элементы/предупреждения на каждый blip; есть потенциал для кэширования форм и снижения количества атрибутов/innerHTML.
- Фильтрация: `updateRadar` многократно приводит строки к lowerCase и создает временные массивы; текстовый поиск идёт поверх нескольких полей. Возможен промежуточный индекс/нормализация.
- Сайдбар: `updateSidebarLists` / `createTechListForSector` активно создают/удаляют DOM, что может быть дорого при частых изменениях фильтров.
- Позиционирование: `avoidRingLabelOverlap` и пересчёт `assignFixedPositionForQuadrant` вызываются для всех элементов; можно кэшировать результаты для неизменных технологий.
