Сравнение страниц Радара технологий

Детальный анализ функциональности страниц для архитекторов и директоров с целью определения оптимального решения для бизнес-заказчика

📊 Обзор приложения "Радар технологий"

Приложение "Радар технологий" — это веб-инструмент для визуализации и анализа технологических решений компании. Оно позволяет:

Основные возможности:
  • Визуализация технологий на интерактивном радаре
  • Фильтрация и поиск технологий по различным критериям
  • Управление данными о технологиях, вендорах и интеграторах
  • Экспорт данных в различные форматы (PDF, Excel)
  • Управление приоритетами технологий
  • Поддержка нескольких предприятий (РМК, ТГОК, МГОК, КБМ, КМЭЗ, АМК)

Две версии страницы радара

В приложении существуют две специализированные версии страницы радара, каждая из которых оптимизирована для своей целевой аудитории:

RMK.html — для архитекторов

Ориентирована на технический анализ и классификацию технологий по стадиям зрелости.

Целевая аудитория: Технические специалисты, архитекторы решений, разработчики

RMK-director.html — для директоров

Ориентирована на принятие управленческих решений о внедрении технологий.

Целевая аудитория: Руководители высшего звена, директора, руководители проектов

👨‍💻 Страница для архитектора (RMK.html)

Назначение и целевая аудитория

Страница для архитекторов предназначена для технических специалистов, которые занимаются анализом, классификацией и оценкой технологий с точки зрения их технических характеристик и зрелости.

Позиционирование технологий на радаре

Метод позиционирования:
  • Позиционирование основано на поле ring из данных технологии
  • Кольца соответствуют стадиям зрелости: Adopt (1), Trial (2), Assess (3), Hold (4)
  • Радиус определяется номером кольца (1-4)
  • Угол определяется квадрантом функционального блока
  • Статическое позиционирование без учета множества факторов

Визуализация технологий

Характеристики визуализации:
  • Разные формы в зависимости от типа технологии:
    • Базовые технологии → треугольник
    • Интегрированные решения → круг
    • Платформенные решения → квадрат
    • Управление с ML и AI → звезда
  • Фиксированный размер всех элементов
  • Подписи колец отображаются на радаре (Adopt, Trial, Assess, Hold)
  • Визуальное различие типов технологий для быстрой идентификации

Управление данными

Доступная функциональность:
  • Добавление новых технологий
  • Редактирование существующих технологий
  • Удаление технологий
  • Базовое управление вендорами (если присутствует в данных)
  • Просмотр информации о технологиях

Дополнительные инструменты анализа

График перспективных технологий (prospects-chart.js):
  • Визуализация технологий на графике с осями
  • Дополнительный инструмент для технического анализа
  • Детальное изучение характеристик технологий
  • Сравнение технологий по различным параметрам

Файл данных

Используется файл enterpriseData.json, который содержит базовую информацию о технологиях без расширенных полей для вендоров и файлов.

Преимущества страницы для архитектора

  • Простота понимания для технических специалистов
  • Прямая связь с классической моделью технологического радара
  • Визуальное различие типов технологий
  • Классическая модель радара с подписями
  • Дополнительный график для детального анализа

Ограничения

  • Не учитывает готовность к внедрению
  • Не отражает приоритетность для бизнеса
  • Статическое позиционирование без учета множества факторов
  • Нет расширенного управления вендорами
  • Нет прикрепления файлов к технологиям
  • Нет управления интеграторами

👔 Страница для директора (RMK-director.html)

Назначение и целевая аудитория

Страница для директоров специально разработана для принятия управленческих решений о внедрении технологий. Она учитывает не только технические характеристики, но и готовность к реализации, наличие вендоров и интеграторов.

Позиционирование технологий на радаре

Математическая модель позиционирования:

Использует логистическую функцию для расчета "дистанции до внедрения" на основе 4 факторов готовности:

  • Технологическая готовность (techRead): 0-3
  • Организационная готовность (organRead): 0-3
  • Покрытие функций (funcCover): 0-3
  • TRL стадия (trlStage): 1-3

Формула расчета:

z_i = Σ(w_k × x_ik) + b
p_i = 1 / (1 + exp(-α × z_i))
r_i = 100 × (1 - p_i)

Технологии с высокой готовностью располагаются ближе к центру радара.

Визуализация технологий

Характеристики визуализации:
  • Все технологии отображаются только кругами — единообразная визуализация
  • Динамический размер круга зависит от количества вендоров:
    • Больше вендоров = больший размер круга
    • Визуально показывает важность технологии для бизнеса
    • Индикация количества доступных вариантов реализации
  • Подписи колец отсутствуют — упрощенная визуализация
  • Фокус на содержании, а не на форме

Управление данными

Расширенная функциональность:
  • Все функции страницы архитектора +
  • Расширенное управление вендорами:
    • Добавление новых вендоров прямо в интерфейсе
    • Автодополнение из списка существующих вендоров
    • Сохранение вендоров в localStorage для последующего использования
    • Множественный выбор вендоров для каждой технологии
  • Управление интеграторами:
    • Привязка интеграторов к вендорам
    • Множественный выбор интеграторов для каждого вендора
    • Отдельное управление интеграторами для каждого вендора
  • Прикрепление файлов:
    • Загрузка файлов к технологиям
    • Хранение информации о файлах в данных технологии
    • Управление списком файлов (добавление, удаление)
    • Поддержка различных типов файлов (документы, презентации, изображения)

Дополнительные инструменты анализа

График перспективных технологий отсутствует

Обоснование: Фокус на принятии решений, а не на техническом анализе. Упрощенный интерфейс для быстрого принятия решений. Основная информация представлена на радаре.

Файл данных

Используется файл enterpriseData-director.json, который содержит расширенную информацию о технологиях с полями vendors и files.

Преимущества страницы для директора

  • Отражает реальную готовность к внедрению
  • Учитывает множественные факторы влияния
  • Позволяет принимать решения на основе объективных метрик
  • Динамическое позиционирование, адаптирующееся к изменениям данных
  • Единообразная визуализация упрощает восприятие
  • Размер круга сразу показывает важность технологии
  • Отсутствие подписей колец снижает визуальный шум
  • Полный контроль над данными о вендорах и интеграторах
  • Возможность прикреплять документы, презентации, контракты к технологиям
  • Более полная картина для принятия решений

Особенности

  • Нет визуального различия типов технологий (но это не критично для директоров)
  • Более сложная модель для понимания (требует объяснения)

Ключевые различия в функциональности

1. Позиционирование технологий

Архитектор: Статическое позиционирование на основе поля ring (1-4), соответствующего стадиям зрелости. Простое и понятное для технических специалистов.

Директор: Динамическое позиционирование на основе математической модели, учитывающей 4 фактора готовности. Технологии с высокой готовностью автоматически перемещаются ближе к центру.

2. Визуализация

Архитектор: Разные формы для разных типов технологий помогают быстро идентифицировать тип. Подписи колец обеспечивают контекст.

Директор: Единообразные круги упрощают восприятие. Размер круга показывает важность (количество вендоров). Отсутствие подписей снижает визуальный шум.

3. Управление данными

Архитектор: Базовое управление технологиями. Ограниченные возможности работы с вендорами.

Директор: Полный контроль над вендорами и интеграторами. Возможность прикреплять файлы (контракты, презентации, спецификации) к технологиям для полной картины при принятии решений.

4. Инструменты анализа

Архитектор: График перспективных технологий для детального технического анализа.

Директор: Фокус на основном радаре с информацией о готовности и вендорах. Упрощенный интерфейс для быстрого принятия решений.

Вердикт: Почему страница для директора более удобна

✅ Ориентация на принятие решений

Позиционирование по готовности к внедрению позволяет сразу видеть приоритетные технологии. Учитывает не только техническую, но и организационную готовность.

✅ Объективность

Математическая модель учитывает множественные факторы и обеспечивает объективную оценку готовности.

✅ Визуальная ясность

Упрощенная визуализация (круги, отсутствие подписей) снижает когнитивную нагрузку. Размер круга показывает важность технологии.

✅ Полнота данных

Расширенное управление вендорами, интеграторами и файлами обеспечивает полную картину для принятия решений.

✅ Адаптивность

Динамическое позиционирование отражает актуальное состояние готовности технологий при изменении данных.
