# Инструкция по активации системы уведомлений

## Шаг 1: Запуск миграций БД

После получения обновленного кода выполните:

```bash
cd backend
python manage.py migrate
```

Это применит создание новых полей и таблиц:
- Добавит поля `editor_notified` и `notification_sent_at` в таблицу `technologies_technologyproposal`
- Создаст новую таблицу `technologies_proposalnotification` для хранения уведомлений

## Шаг 2: Перезагрузка сервера

Перезагрузите Django-сервер:

```bash
# Если используется встроенный dev-сервер
python manage.py runserver

# Или перезагрузите Gunicorn/другой production сервер
kill -HUP $(cat /path/to/gunicorn.pid)  # для gunicorn с pidfile
```

## Шаг 3: Обновить браузер

1. Откройте приложение в браузере
2. Выполните `Ctrl+F5` (жесткая перезагрузка) для очистки кэша
3. Перезагрузитесь в системе для обновления конфигурации

## Тестирование

### Быстрая проверка:

1. **Логинтесь как редактор (Editor):**
   - Создайте предложение на добавление/изменение/удаление технологии
   - Отправьте на модерацию

2. **Логинтесь как администратор (Owner/Admin):**
   - Откройте панель модерации
   - Найдите предложение от редактора
   - Одобрите/отклоните/отложите его

3. **Вернитесь как редактор:**
   - Обновите страницу или откройте панель модерации
   - **Должно появиться всплывающее уведомление** в правом верхнем углу первое

Если уведомления не появляются, смотрите раздел "Диагностика" ниже.

## Структура данных уведомления

Каждое уведомление содержит:
- `id` - уникальный идентификатор
- `notification_type` - тип уведомления (approved/rejected/postponed)
- `title` - заголовок (например, "Предложение одобрено")
- `message` - полный текст с объяснением
- `is_read` - прочитано ли уведомление
- `created_at` - дата создания
- `read_at` - дата и время прочтения (если прочитано)
- `proposal_action` - тип действия (create/update/delete)
- `proposal_status` - статус предложения
- `technology_name` - название технологии
- `reviewer_username` - имя администратора, который принял решение

## Диагностика

### Проблема: Уведомления не появляются

**Проверка 1 - Миграции:**
```bash
cd backend
python manage.py showmigrations technologies
# Должно показать галку напротив 0004_proposal_notifications
```

**Проверка 2 - Console браузера:**
- F12 → Console (Консоль)
- Посмотрите на ошибки (красные сообщения)
- Часто ошибка: "404 Not Found" при запросе к `/api/v1/proposal-notifications`

**Проверка 3 - API напрямую:**
```bash
# В другом терминале, с авторизованной сессией
curl -H "Authorization: Bearer <your-token>" \
     http://localhost:8000/api/v1/proposal-notifications
```

Должен вернуться JSON массив с уведомлениями.

**Проверка 4 - База данных:**
```bash
# Подключиться к БД
python manage.py dbshell

# Посмотреть таблицы
.tables

# Посмотреть уведомления
SELECT * FROM technologies_proposalnotification ORDER BY created_at DESC LIMIT 5;
```

### Проблема: "Не удалось загрузить уведомления"

Это означает, что фронтэнд не смог подключиться к API.

**Решение:**
1. Проверьте, запущен ли backend-сервер
2. Проверьте, правильный ли URL в dataService (должен быть `/api/v1`)
3. Проверьте консоль браузера на ошибки CORS (если фронтэнд на другом домене)

### Проблема: Уведомление создается, но не отправляется редактору

**Причины:**
1. Редактор не загружает страницу после одобрения
2. JS ошибка при загрузке уведомлений

**Решение:**
1. Убедитесь, что редактор обновил страницу или открыл панель модерации ПОСЛЕ одобрения
2. Проверьте консоль браузера на ошибки
3. Перезагрузите страницу вручную

## О чем нужно помнить

1. **Уведомления отправляются только редактору**, создавшему предложение
2. **Владелец видит только свой успех** в виде уведомления о том, что предложение обработано
3. **Уведомления не удаляются автоматически** - они накапливаются в БД (макс 100 на странице)
4. **Всплывающее уведомление закрывается** при клике на `×` и отмечается как прочитанное

## Откат изменений (если нужно)

Если нужно откатить эти изменения:

```bash
cd backend
python manage.py migrate technologies 0003
# Это откатит к версии до 0004_proposal_notifications
```

Но учтите, что все данные в таблице `technologies_proposalnotification` будут потеряны.
