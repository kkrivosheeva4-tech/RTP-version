Добавление шрифта DejaVu Sans для корректной генерации PDF с кириллицей
Почему

---

jsPDF по умолчанию использует встроенные шрифты, которые не поддерживают кириллицу. Чтобы PDF отображал русский текст корректно, нужно добавить TTF-шрифт (например, DejaVu Sans) и позволить скрипту встроить его.

## Локальный способ

1. Скачайте DejaVu Sans (DejaVuSans.ttf) с официального источника: https://www.fontsquirrel.com/fonts/dejavu-sans или https://dejavu-fonts.github.io/
2. Поместите файл `DejaVuSans.ttf` в папку `fonts/` рядом с `RMK-director.html`.

## Автоматическая загрузка (PowerShell)

Если вы хотите загрузить шрифт автоматически, выполните из корня проекта:

powershell -ExecutionPolicy Bypass -File .\fonts\download_dejavu.ps1

## CDN (быстрое решение)

Сценарий экспорта теперь пробует загрузить шрифт с CDN (jsDelivr):

https://cdn.jsdelivr.net/gh/dejavu-fonts/dejavu-fonts@latest/ttf/DejaVuSans.ttf

Этот CDN обычно предоставляет корректные CORS-заголовки, поэтому браузерный fetch будет работать. Если у вас есть подключение к сети, скрипт автоматически попытается скачать шрифт с CDN.

## Встроенный base64 (альтернатива)

Если вы не хотите хранить бинарный файл в репозитории, можно положить base64-строку шрифта в глобальную переменную `window.DEJAVU_FONT_BASE64` перед загрузкой `RMK2.js`.

Пример (условный):

<script>
  // VERY LARGE STRING — пример здесь не приводится.
  window.DEJAVU_FONT_BASE64 = 'AAEAAA...';
</script>

## Поведение скрипта

1. Попытка использовать `window.DEJAVU_FONT_BASE64` (если задано).
2. Попытка загрузить локальный файл из `./fonts/DejaVuSans.ttf`, `./DejaVuSans.ttf` или `/fonts/DejaVuSans.ttf`.
3. Попытка загрузить шрифт с jsDelivr CDN.
4. Если все попытки провалились, скрипт уведомит пользователя и применит транслитерацию как запасной вариант, чтобы текст остался читаемым.

## Лицензия

DejaVu Sans распространяется под лицензией Bitstream Vera / public domain-like (OFL для некоторых патчей). Убедитесь, что использование шрифта в вашем проекте соответствует требованиям лицензии.
