## 🎯 Описание проекта

«Радар технологий» — это интерактивный инструмент для визуализации распределения и зрелости технологий внутри организации. Приложение позволяет:

- Визуализировать технологический ландшафт в виде интерактивного радара
- Анализировать цифровую зрелость по различным предприятиям
- Управлять пользователями и отслеживать активность через журнал аудита
- Экспортировать данные для дальнейшего анализа

## ✨ Основные возможности

### 1. Визуализация радара технологий
- Интерактивный радар с технологиями, распределенными по квадрантам и кольцам зрелости
- Фильтрация по функциональным блокам, функциям, типам технологий и статусам
- Поиск технологий по названию
- Детальная информация о каждой технологии при клике
- Поддержка темной и светлой темы

### 2. Аналитика
- Статистика по холдингу и отдельным предприятиям
- Графики и диаграммы для визуализации данных
- Фильтрация по периодам времени
- Экспорт отчетов в PDF и Excel

### 3. Административная панель
- Управление пользователями (создание, редактирование, удаление)
- Журнал аудита всех действий в системе
- Создание и управление резервными копиями
- Экспорт данных пользователей и журнала аудита

### 4. Система авторизации
- Безопасный вход в систему
- Разграничение прав доступа по ролям
- Запоминание сессии пользователя

## 🔧 Требования

- **Веб-браузер**: Chrome, Firefox, Edge, Safari (последние версии)
- **Веб-сервер**: Python 3, PHP или PowerShell (для локального запуска)
- **Интернет**: требуется для загрузки CDN библиотек (jsPDF, html2canvas, Chart.js)

> ⚠️ **Важно**: Node.js **НЕ требуется**! Проект полностью работает без Node.js. Все зависимости загружаются через CDN.

## 🚀 Установка и запуск

### ⚡ Быстрый старт (рекомендуется)

#### Windows:
1. Дважды кликните на файл `start-server.bat`
2. Выберите способ запуска (Python рекомендуется)
3. Откройте браузер: `http://localhost:8000`

#### Linux/Mac:
```bash
chmod +x start-server.sh
./start-server.sh
```

### 📖 Подробные инструкции

См. файл **[ЗАПУСК_БЕЗ_NODE.md](ЗАПУСК_БЕЗ_NODE.md)** для полной инструкции по запуску без Node.js.

### Вариант 1: Python HTTP сервер (рекомендуется)

```bash
# Python 3
python -m http.server 8000

# Или Python 2
python -m SimpleHTTPServer 8000
```

Затем откройте в браузере: `http://localhost:8000`

### Вариант 2: PHP Built-in Server

```bash
php -S localhost:8000
```

### Вариант 3: Live Server (VS Code)

1. Установите расширение "Live Server" в VS Code
2. Откройте папку проекта в VS Code
3. Правой кнопкой на `index.html` → "Open with Live Server"

### ⚠️ Важно!

**НЕ открывайте HTML файлы напрямую** (двойным кликом), так как:
- Браузер блокирует загрузку JSON файлов из-за политики безопасности CORS
- Некоторые функции могут не работать

**Всегда используйте локальный веб-сервер!**

### Установка зависимостей (опционально)

> **Примечание**: Установка зависимостей через npm **не требуется** для работы проекта. Все библиотеки загружаются через CDN автоматически.

Если вы хотите использовать локальные версии библиотек (для разработки без интернета):

```bash
npm install
```

Зависимости проекта (используются через CDN, не требуют установки):
- `d3` (^7.9.0) - библиотека для визуализации данных (в проекте не используется)

## 📖 Использование

### Первый запуск

1. Откройте приложение в браузере
2. На главной странице выберите предприятие из навигационного меню (например, "РМК")
3. Для доступа к административным функциям необходимо авторизоваться

### Авторизация

1. Перейдите на страницу авторизации (`auth.html`) или используйте кнопку входа в системе
2. Введите учетные данные:

**Тестовые учетные записи:**
- **Администратор**:
  - Логин: `admin`
  - Пароль: `admin123`
- **Архитектор**:
  - Логин: `architect`
  - Пароль: `architect123`
- **Гость**:
  - Логин: `guest`
  - Пароль: `guest123`

3. При необходимости установите флажок "Запомнить меня"
4. Нажмите "Войти"

### Работа с радаром технологий

1. **Выбор предприятия**: Нажмите на кнопку предприятия в верхней навигации
2. **Поиск технологий**: Используйте поле поиска в боковой панели
3. **Фильтрация**:
   - Откройте панель фильтров (кнопка с иконкой фильтра)
   - Выберите нужные критерии:
     - Функциональные блоки
     - Функции
     - Тип технологий
     - Статус зрелости
4. **Просмотр деталей**: Кликните на технологию на радаре для просмотра подробной информации
5. **Экспорт**: Используйте кнопки экспорта для сохранения данных

### Работа с аналитикой

1. Перейдите на страницу "Аналитика" из навигации
2. Выберите предприятие и период времени
3. Просмотрите статистику и графики
4. Экспортируйте отчеты в PDF или Excel при необходимости

### Административная панель

**Доступ**: Только для пользователей с ролями "admin" или "architect"

#### Управление пользователями

1. Перейдите в раздел "Пользователи"
2. Используйте поиск и фильтры для нахождения нужных пользователей
3. **Добавление пользователя**: Нажмите кнопку "Добавить пользователя"
4. **Редактирование**: Нажмите иконку редактирования рядом с пользователем
5. **Удаление**: Нажмите иконку удаления (требуется подтверждение)

#### Журнал аудита

1. Перейдите в раздел "Журнал аудита"
2. Используйте фильтры:
   - Диапазон дат
   - Пользователь
   - Тип действия
3. Экспортируйте данные при необходимости

#### Резервные копии

1. Перейдите в раздел "Резервные копии"
2. **Создание копии**: Нажмите "Создать резервную копию"
3. **Восстановление**: Нажмите кнопку восстановления рядом с нужной копией
4. **Удаление**: Нажмите кнопку удаления (требуется подтверждение)

## 👥 Роли пользователей

### Гость (guest)

## 🛠 Технические детали

### Технологии

- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **Визуализация**: Chart.js (через CDN)
- **Экспорт**: jsPDF, html2canvas (через CDN)
- **Хранение данных**: LocalStorage, JSON файлы
- **Зависимости**: Все библиотеки загружаются через CDN, Node.js не требуется

### Архитектура

- **Модульная структура**: Каждая страница имеет свой JavaScript и CSS файл
- **Организация файлов**: Стандартная веб-архитектура с разделением на `assets/` (статические ресурсы), `src/` (исходный код), `scripts/` (утилиты)
- **Общие компоненты**: `src/css/common.css` для общих стилей
- **Данные**: JSON файлы в директории `src/data/ru/`
- **Темы**: Поддержка светлой и темной темы через CSS переменные

### Структура данных

Данные хранятся в JSON формате:

- `enterpriseData.json` - данные о предприятиях и технологиях
- `bloks.json` - функциональные блоки
- `functions.json` - функции
- `sector.json` - секторы/квадранты
- `status.json` - статусы зрелости
- `techTypes.json` - типы технологий

### Безопасность

- Авторизация через localStorage (для демо-версии)
- Проверка прав доступа на каждой странице
- Журналирование всех действий пользователей
- Валидация входных данных

## 🔨 Разработка

### Добавление новых технологий

1. Откройте файл `src/data/ru/enterpriseData.json`
2. Добавьте новую запись в массив технологий
3. Убедитесь, что указаны все необходимые поля:
   - `name` - название технологии
   - `quadrant` - квадрант (0-3)
   - `ring` - кольцо зрелости (0-3)
   - `techType` - тип технологии
   - `block` - функциональный блок
   - и другие поля

### Изменение стилей

- Общие стили: `src/css/common.css`
- Стили конкретной страницы: соответствующий CSS файл в `src/css/`
- Темы: CSS переменные в `:root` селекторах

### Добавление нового предприятия

1. Добавьте кнопку в навигацию (`enterprise-nav`)
2. Обновите данные в `src/data/ru/enterpriseData.json`
3. При необходимости создайте отдельную страницу по аналогии с `src/pages/RMK.html`

## 📝 Примечания

- Приложение использует localStorage для хранения данных пользователей и настроек
- Для продакшн-версии рекомендуется подключить backend API
- Все данные в текущей версии хранятся локально в браузере
- Резервные копии создаются в памяти и не сохраняются на сервере

## 🐛 Решение проблем

### Проблема: Не загружаются данные

**Решение**: Убедитесь, что вы используете локальный веб-сервер, а не открываете файлы напрямую.

### Проблема: Не работает авторизация

**Решение**:
1. Очистите localStorage браузера
2. Проверьте правильность ввода учетных данных
3. Убедитесь, что JavaScript включен в браузере

### Проблема: Графики не отображаются

**Решение**:
1. Проверьте подключение к интернету (используются CDN библиотеки)
2. Откройте консоль браузера (F12) для просмотра ошибок
3. Обновите страницу
