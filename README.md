## 🎯 Описание проекта

«Радар технологий» — это интерактивный инструмент для визуализации распределения и зрелости технологий внутри организации. Приложение позволяет:

- Визуализировать технологический ландшафт в виде интерактивного радара
- Анализировать цифровую зрелость по различным предприятиям
- Управлять пользователями и отслеживать активность через журнал аудита
- Экспортировать данные для дальнейшего анализа

## ✨ Основные возможности

### 1. Визуализация радара технологий

- Интерактивный радар с технологиями, распределенными по квадрантам и кольцам зрелости
- Фильтрация по функциональным блокам, функциям, типам технологий и статусам
- Поиск технологий по названию
- Детальная информация о каждой технологии при клике
- Поддержка темной и светлой темы

### 2. Административная панель

- Управление пользователями (создание, редактирование, удаление)
- Журнал аудита всех действий в системе
- Создание и управление резервными копиями
- Экспорт данных пользователей и журнала аудита

### 3. Система авторизации

- Безопасный вход в систему
- Двухфакторная аутентификация (2FA) для всех ролей
- Разграничение прав доступа по ролям
- Запоминание сессии пользователя

## 🔧 Требования

- **Веб-браузер**: Chrome, Firefox, Edge, Safari (последние версии)
- **Backend/запуск приложения**: Python 3.14+ (Django)
- **Интернет**: требуется для загрузки CDN библиотек (`jsPDF`, `html2canvas`, `Chart.js`)

### Модель запуска: Django runtime

**Приложение в runtime работает на Django.** Основной локальный запуск и backend runtime не требуют отдельной frontend-сборки.

Сервер HTTP — это Django (`manage.py runserver` или WSGI в production). Фронтенд отдается через Django templates/staticfiles.

Production WSGI baseline:

```bash
gunicorn --config gunicorn.conf.py config.wsgi:application
```

## 📖 Использование

### Первый запуск

1. Откройте приложение в браузере
2. На главной странице выберите предприятие из навигационного меню (например, "РМК")
3. Для доступа к административным функциям необходимо авторизоваться

### Авторизация

1. Перейдите на страницу авторизации (`auth.html`) или используйте кнопку входа в системе
2. Введите учетные данные:

**Тестовые учетные записи:**

- **Администратор**:
  - Логин: `admin`
  - Пароль: `admin123`
- **Архитектор**:
  - Логин: `architect`
  - Пароль: `architect123`

3. При необходимости установите флажок "Запомнить меня"
4. Нажмите "Войти"
5. После входа потребуется двухфакторная аутентификация (2FA): при первом входе — настройка через QR-код и код из приложения; при последующих — ввод 6-значного кода.

### Работа с радаром технологий

1. **Выбор предприятия**: Нажмите на кнопку предприятия в верхней навигации
2. **Поиск технологий**: Используйте поле поиска в боковой панели
3. **Фильтрация**:
   - Откройте панель фильтров (кнопка с иконкой фильтра)
   - Выберите нужные критерии:
     - Функциональные блоки
     - Функции
     - Тип технологий
     - Статус зрелости
4. **Просмотр деталей**: Кликните на технологию на радаре для просмотра подробной информации
5. **Экспорт**: Используйте кнопки экспорта для сохранения данных

### Основная страница радара (radar.html)

**Доступ**: Только для пользователей с ролями "director" или "project_manager"

**Особенности**:

- **Модифицированный радар**:
  - Позиционирование технологий по готовности к реализации и внедрению
  - Размер кругов зависит от количества вендоров
  - Отсутствуют подписи колец
  - Все технологии отображаются только в виде кругов
- **Расширенные формы**:
  - Управление вендорами и интеграторами
  - Прикрепление файлов к технологиям
  - Подсказки для всех полей
  - Переименованные TRL оценки (1-Исследовательская, 2-Прототип, 3-Технология готова к внедрению)
- **Отдельный файл данных**: `enterpriseData-director.json` для хранения данных директорской страницы

### Административная панель

**Доступ**: Только для пользователей с ролью "admin"

#### Управление пользователями

1. Перейдите в раздел "Пользователи"
2. Используйте поиск и фильтры для нахождения нужных пользователей
3. **Добавление пользователя**: Нажмите кнопку "Добавить пользователя"
4. **Редактирование**: Нажмите иконку редактирования рядом с пользователем
5. **Удаление**: Нажмите иконку удаления (требуется подтверждение)

#### Журнал аудита

1. Перейдите в раздел "Журнал аудита"
2. Используйте фильтры:
   - Диапазон дат
   - Пользователь
   - Тип действия
3. Экспортируйте данные при необходимости

#### Резервные копии

1. Перейдите в раздел "Резервные копии"
2. **Создание копии**: Нажмите "Создать резервную копию"
3. **Восстановление**: Нажмите кнопку восстановления рядом с нужной копией
4. **Удаление**: Нажмите кнопку удаления (требуется подтверждение)

- Все права архитектора
- Доступ к административной панели
- Управление пользователями
- Просмотр журнала аудита
- Полный доступ ко всем функциям системы
- Управление резервными копиями
- Экспорт всех данных

## 🛠 Технические детали

### Технологии

- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **Визуализация**: Chart.js (через CDN)
- **Экспорт**: jsPDF, html2canvas (через CDN)
- **Хранение данных**: PostgreSQL, Django API, JSON-справочники и Django staticfiles
- **Runtime**: Python 3.14+, Django 6, gunicorn
- **Зависимости**: jsPDF, html2canvas, Chart.js (через CDN для radar)

### Архитектура

- **Модульная структура**:
  - Каждая страница имеет свой JavaScript и CSS файл
  - JavaScript модули организованы в `src/js/modules/` по функциональным категориям:
    - **core/** - Основные модули системы (управление состоянием, загрузка данных, обработка ошибок)
    - **ui/** - Модули пользовательского интерфейса (фильтры, формы, модальные окна, сайдбар)
    - **radar/** - Модули визуализации радара (рендеринг, позиционирование, квадранты)
    - **business/** - Бизнес-логика (авторизация, экспорт, приоритеты)
    - **integration/** - Интеграционные модули и утилиты
  - Подробная документация по всем модулям: **[docs/MODULES_DOCUMENTATION.md](docs/MODULES_DOCUMENTATION.md)**
- **Организация файлов**: Стандартная веб-архитектура с разделением на `assets/` (статические ресурсы), `src/` (исходный код), `scripts/` (утилиты)
- **Общие компоненты**: `src/css/common.css` для общих стилей
- **Данные**: JSON файлы в директории `src/data/ru/`
- **Темы**: Поддержка светлой и темной темы через CSS переменные
- **Управление состоянием**: Централизованное через StateManager с системой подписок
- **Производительность**: Оптимизации через DOMCache, RenderQueue, мемоизацию

### Структура данных

Данные хранятся в JSON формате:

- `enterpriseData.json` - данные о предприятиях и технологиях (для основной страницы)
- `enterpriseData-director.json` - данные для директорской страницы (с полями vendors и files)
- `blocks.json` - функциональные блоки
- `functions.json` - функции
- `sector.json` - секторы/квадранты
- `status.json` - статусы зрелости
- `techTypes.json` - типы технологий

**Расширенная структура данных для директорской страницы**:
Технологии в `enterpriseData-director.json` могут содержать дополнительные поля:

- `vendors` - массив вендоров с интеграторами
- `files` - массив прикрепленных файлов

### Безопасность

- Авторизация через localStorage (для демо-версии)
- Проверка прав доступа на каждой странице
- Журналирование всех действий пользователей
- Валидация входных данных

## 🔨 Разработка

### Модульная архитектура JavaScript

Проект использует модульную архитектуру JavaScript для обеспечения:

- Разделения ответственности между модулями
- Переиспользуемости кода
- Легкости тестирования и поддержки
- Хорошей производительности

**Документация по модулям**: См. файл **[docs/MODULES_DOCUMENTATION.md](docs/MODULES_DOCUMENTATION.md)** для подробного описания всех JavaScript модулей, их функций и взаимосвязей.

**Основные принципы работы с модулями:**

1. Всегда используйте `StateManager` для хранения состояния приложения
2. Используйте `DOMCache` вместо прямых DOM запросов
3. Используйте `EventManager` для делегирования событий
4. Используйте `RenderQueue` для батчинга DOM обновлений
5. Используйте `ErrorHandler` для централизованной обработки ошибок

### Добавление новых технологий

1. Откройте файл `src/data/ru/enterpriseData.json`
2. Добавьте новую запись в массив технологий
3. Убедитесь, что указаны все необходимые поля:
   - `name` - название технологии
   - `quadrant` - квадрант (0-3)
   - `ring` - кольцо зрелости (0-3)
   - `techType` - тип технологии
   - `block` - функциональный блок
   - и другие поля

### Изменение стилей

- Общие стили: `src/css/common.css`
- Стили конкретной страницы: соответствующий CSS файл в `src/css/`
- Темы: CSS переменные в `:root` селекторах

### Добавление нового предприятия

1. Добавьте кнопку в навигацию (`enterprise-nav`)
2. Обновите данные в `src/data/ru/enterpriseData.json`
3. При необходимости создайте отдельную страницу по аналогии с `src/pages/radar.html`

## 🐛 Решение проблем

### Проблема: Не загружаются данные

**Решение**: Убедитесь, что вы используете локальный веб-сервер, а не открываете файлы напрямую.

### Проблема: Не работает авторизация

**Решение**:

1. Очистите localStorage браузера
2. Проверьте правильность ввода учетных данных
3. Убедитесь, что JavaScript включен в браузере
