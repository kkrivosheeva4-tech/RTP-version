# Система уведомлений редакторам о результатах модерации

## Проблема
Редактор отправляет предложение на изменение технологии, но не получает обратную связь о том, было ли его предложение принято, отклонено или отложено администратором/владельцем.

## Решение
Реализована полная система уведомлений, которая:
1. Сохраняет уведомление в БД при модерации предложения
2. Отправляет уведомление редактору через API
3. Отображает уведомление в браузере редактора в виде всплывающего сообщения

## Затронутые файлы

### Backend (Python/Django)

#### 1. Models (`backend/technologies/models.py`)
- **Добавлены поля в `TechnologyProposal`:**
  - `editor_notified` (BooleanField) - отметка о том, отправлено ли уведомление
  - `notification_sent_at` (DateTimeField) - время отправки уведомления

- **Новая модель `ProposalNotification`:**
  - Хранит все уведомления для редакторов
  - Связь с `TechnologyProposal` и пользователем (редактором)
  - Поля: `notification_type` (approved/rejected/postponed), `title`, `message`, `is_read`, `created_at`, `read_at`

#### 2. Миграции (`backend/technologies/migrations/`)
- **Файл: `0004_proposal_notifications.py`**
  - Добавляет новые поля в `TechnologyProposal`
  - Создает новую таблицу `ProposalNotification`

#### 3. Serializers (`backend/technologies/serializers.py`)
- **`TechnologyProposalSerializer`** - обновлен для работы с новыми полями
- **`ProposalNotificationSerializer`** - новый сериализатор для уведомлений
  - Включает информацию о предложении, типе действия и результате модерации

#### 4. Views (`backend/technologies/views.py`)
- **Функция `_create_editor_notification()`** - создает уведомление редактору
  - Вызывается при одобрении, отклонении или отложении предложения
  - Создает осмысленное сообщение на основе типа действия и решения

- **Обновлены три API контроллера:**
  - `TechnologyProposalApproveAPIView` - одобрение + отправка уведомления
  - `TechnologyProposalRejectAPIView` - отклонение + отправка уведомления
  - `TechnologyProposalPostponeAPIView` - отложение + отправка уведомления

- **Новый API контроллер `EditorProposalNotificationsAPIView`:**
  - GET `/api/v1/proposal-notifications` - получить все уведомления текущего пользователя
  - PATCH `/api/v1/proposal-notifications` - отметить уведомление как прочитанное

#### 5. API URLs (`backend/config/api_urls.py`)
- Добавлены маршруты для нового endpoint:
  - `POST /api/v1/proposal-notifications` - получить уведомления
  - `PATCH /api/v1/proposal-notifications` - обновить статус прочтения

### Frontend (JavaScript)

#### 1. Data Service (`src/js/modules/core/data-service.js`)
- **Новая API функция `apiLoadProposalNotifications()`**
  - GET запрос к `/api/v1/proposal-notifications`
  - Загружает все уведомления для текущего пользователя

- **Новая API функция `apiMarkProposalNotificationAsRead()`**
  - PATCH запрос для отметки уведомления как прочитанного

- **Методы в объекте `DataService`:**
  - `loadProposalNotifications()` - асинхронная загрузка уведомлений
  - `markProposalNotificationAsRead()` - отметить как прочитанное

#### 2. Moderation Module (`src/js/modules/business/moderation.js`)
- **Новая переменная глобального состояния:**
  - `latestProposalNotifications` - хранит загруженные уведомления

- **Обновлена функция `refreshPanel()`:**
  - Теперь загружает уведомления вместе с предложениями
  - Вызывает `displayProposalNotifications()` для отображения

- **Новая функция `displayProposalNotifications()`:**
  - Создает в браузере всплывающие уведомления
  - Располагаются в верхнем правом углу экрана
  - Цвет границы зависит от типа решения:
    - 🟢 Зеленый - одобрено (approved)
    - 🔴 Красный - отклонено (rejected)
    - 🟠 Оранжевый - отложено (postponed)
  - Могут быть закрыты редактором (кнопка ×)
  - Автоматически отмечаются как прочитанные при закрытии

## Процесс работы

### 1. Редактор создает предложение
```
Редактор → createProposal() → Backend: POST /api/v1/technology-proposals
Backend сохраняет proposal со статусом "draft"
```

### 2. Владелец/Администратор модерирует предложение
```
Администратор → reviewProposal(id, 'approve'/'reject'/'postpone')
↓
Backend: POST /api/v1/technology-proposals/{id}/approve (или reject/postpone)
↓
Backend:
  1. Применяет изменения (создает/обновляет/удаляет технологию)
  2. Вызывает _create_editor_notification()
  3. Сохраняет ProposalNotification в БД
  4. Возвращает обновленное предложение
```

### 3. Редактор получает уведомление
```
Редактор открывает/обновляет панель модерации
↓
refreshPanel()
↓
loadProposalNotifications() → GET /api/v1/proposal-notifications
↓
Backend возвращает список уведомлений для редактора
↓
displayProposalNotifications() показывает всплывающие сообщения
```

## Установка и запуск

### 1. Применить миграции
```bash
cd backend
python manage.py migrate
```

### 2. Перезагрузить сервер
```bash
# Если используется development сервер
python manage.py runserver

# Или для production с Gunicorn
gunicorn config.wsgi --reload
```

### 3. Обновить страницу браузера
После применения миграций и перезагрузки сервера обновите страницу приложения.

## Тестирование

### Для ручного тестирования:

1. **Создать редактора:**
   - Пользователь с ролью "Editor"

2. **Создать администратора:**
   - Пользователь с ролью "Owner" или "Admin"

3. **Редактор создает предложение:**
   - Нажимает на иконку предложения
   - Заполняет данные
   - Отправляет на модерацию

4. **Администратор модерирует:**
   - Входит в панель модерации
   - Видит предложение в списке "На модерацию"
   - Одобряет/отклоняет/откладывает

5. **Редактор получает уведомление:**
   - Обновляет страницу или открывает панель
   - Видит всплывающее уведомление в правом верхнем углу

## Проблемы и решения

### Если уведомления не появляются:

1. **Проверить миграции:**
   ```bash
   python manage.py showmigrations
   # Должно показать 0004_proposal_notifications как применённую
   ```

2. **Проверить консоль браузера:**
   - F12 → Console
   - Проверить нет ли ошибок при загрузке уведомлений

3. **Проверить API:**
   ```bash
   curl -H "Authorization: Bearer <token>" \
        http://localhost:8000/api/v1/proposal-notifications
   ```

4. **Проверить БД:**
   ```bash
   python manage.py dbshell
   SELECT * FROM technologies_proposalnotification;
   ```
